# Phaserローカルログ

Vite開発サーバーは、開発診断用の詳細ランJSONをこのフォルダへ保存します。

- 手動ラン: `runs/*.json`、最新200件。
- 手動ラン一覧: `runs/summary.tsv`と`runs/summary.csv`。
- デバッグラン: `debug/*.json`、最新100件。
- 明示的な自動テスト: `tests/*.json`、最新20件。
- JSONファイルはgit管理外です。
- `npm run dev`で起動し、1秒以上のランがゲームオーバーになると起点別に保存します。
- 通常の自動ブラウザテストと1秒未満のランは、自動保存しません。
- プレイヤー向け履歴とランキングは版付きブラウザ保存を使い、このフォルダを読みません。
- 一覧ファイルは開発サーバー起動時と手動ラン保存時にJSONから再生成します。
- `summary.tsv`はタブ区切りです。全体をコピーすると、表計算ソフトへセル分割して貼り付けられます。
- v0.6.4ではEXサイクル、自動EX取得数、敵の直接追跡・迂回・フォールバック、経路場構築回数、迂回比率、障害物・外周・外周面別の反射数を一覧へ含めます。
- v0.6.5では障害物 / 外周別の反射後命中・撃破、外周面別の反射後命中、実フレーム標本数、平均 / p95 / 最大フレーム時間、50ms以上のフレーム数、Phaser実測FPSをJSONと一覧へ含めます。
- v0.6.5は計装だけを追加し、`rulesetVersion`は`phaser-v0.6.4-pulse-ballistics`を維持します。
- v0.7では遠征の勝敗・到達Act・カード進行、Commander / Charger、ボス残HP・攻撃別被弾を一覧へ含めます。

容量確認:

```bash
du -sh logs
find logs -maxdepth 2 -type f -name '*.json' -printf '%p %s bytes\n'
```

保持上限を超えた古いファイルは、次の保存時に起点別で削除されます。
