## 目的

観戦AIは、武器、敵、成長、後半圧力を同じ条件で観察するための決定論的な検証プレイヤーです。ゲーム本体へ能力補正を加えず、手動プレイと同じ`InputSnapshot`だけを生成します。AIランはデバッグランとして記録し、ランキング対象にはしません。

主な用途は次の2つです。

| プロファイル | 用途 | 武器差を許す範囲 |
| --- | --- | --- |
| `fair` | 武器バランスの比較 | 照準、適正距離、射撃機会、武器固有効果だけ |
| `ceiling` | 各武器の技能上限と観戦デモ | 上記に加え、武器別の回復開始と安全余白 |

`fair`では開放空間、緊急HP、敵弾、接触、遠距離射線の生存係数を全武器で共通化します。将来、人間プレイヤーとの比較が必要になった場合は、可視情報だけを使い100から180msの反応遅延を持つ`human-reference`を別プロファイルとして追加します。

対象外:

- 学習済みモデルによる人間らしさの再現。
- AI専用の速度、攻撃力、被ダメージ軽減。
- 観戦結果だけを根拠にしたゲーム本体の武器調整。
- AIランの通常ランキング登録。

## モジュール境界

`WorldState`を書き換える機能は`stepWorld`だけに残し、AIは読み取り専用フレームから判断を返します。

| 境界 | 責務 |
| --- | --- |
| `AutoPilotAgent` | 前回移動、照準対象、戦術コミットを保持し、最終Decisionを合成する |
| `AutoPilotPolicy` | 回復、XP、短時間撃破、射線確保、巡回のUtilityを比較する |
| `AutoPilotTargeting` | 標的、迎撃点、Spread扇形、射撃可否を決める |
| `AutoPilotThreat` | 相対速度、敵経路、連続TTC、予測ダメージ、正規化リスクを計算する |
| `AutoPilotMotionPlanner` | 安全包絡の中から短期速度を選び、戦術Intentを実行する |
| `AutoPilotNavigationPort` | 直線可視性、rot.js Dijkstra、経路長、waypointを隠蔽する |
| `AutoPilotProbe` | 固定シード実行、行動比率、回収品質、武器成果、p95を集計する |

依存方向は`Agent -> Policy / Targeting / Motion -> Threat / Navigation Port`です。Phaser Scene、DOM、描画、保存処理には依存しません。

## 共通危険評価

敵弾はプレイヤーとの相対位置`p`と相対速度`v`を使います。`dot(p, v) >= 0`で重なっていなければ離反中なので、将来リスクを与えません。接近中は二次方程式で円同士の衝突時刻を解き、サンプル間を高速で横切る弾も検出します。

敵は0.20、0.42、0.62、0.82秒先まで区分予測します。直線が障害物で塞がれる場合は、実ゲームと同じDijkstra経路場の進行方向を使います。候補速度ごとに次を0から1へ正規化します。

- 予測ダメージと現在のダメージ許容量。
- 最小TTC。
- 敵弾と接触敵の最近接距離。
- 8方向のうち残る逃走経路数。

同じ計算器を静止時の姿勢、候補速度、デバッグ表示へ使い、閾値だけが別実装にならないようにします。

## 局所速度探索

毎フレーム、停止、戦術方向、前回方向、16方位を0.58秒先まで評価します。安全候補の最良方向には周辺角度を2本追加します。フル速度では障害物を越える場合、移動候補が残らない場合、またはwaypointが予測区間内にある場合だけ、戦術方向と前回方向へ0.5倍と0.25または0.75倍の候補を追加します。

選択順序:

1. 障害物横断、アリーナ外、崩壊領域外を除外する。
2. 予測衝突数が最小の候補だけを残す。
3. 最小リスク`Rmin`を求め、`R <= Rmin + 0.03 + 0.12(1 - Rmin)^2`の安全包絡だけを残す。
4. 安全包絡内で回収進捗、射線、適正距離、開放空間、継続方向を比較する。

危険と目的を単純な加重和だけで相殺しないため、XPや射線の利益が回避可能な直近衝突を上回ることはありません。Movementが戦術Intentを拒否した場合もIntent自体は保持し、`projectileCollision`、`projectileThreat`、`enemyCollision`、`enemyThreat`を上書き理由として記録します。さらに`progress`、`goalReached`、`magnetWait`、`safetyDeflection`、`safetyStop`、`noSafeVelocity`、`pathBlocked`を`motionDisposition`へ出し、安全上の停止と実装上の硬直を分離します。

Pickupが磁力圏外にあり、最良候補が停止でも、同じ安全包絡に移動候補が残る場合は停止を採用しません。まず目標方向へ進む候補、次に最良の移動候補を選びます。これは満HP回復の回避ペナルティが複数重なり、XP回収表示のまま停止し続けるデッドロックを防ぐためです。安全包絡に移動候補がない場合と、Pickupの吸着待ちでは停止を許可します。

## 戦術コミット

初期コミット時間はXP 0.45秒、回復0.72秒、交戦0.30秒です。対象消滅、期限切れ、経路不能、回収経路が0.35秒短くならない場合は解除します。停滞対象は0.9秒候補から除外し、同じ地点への即時再コミットを防ぎます。直近衝突はコミットを破棄せず、Movementが即時回避します。

切替には現在目標より高いUtilityが必要です。

- 同じ行動系: 12%以上かつ絶対差0.08以上。
- 回収と交戦をまたぐ切替: 18%以上かつ絶対差0.12以上。
- 緊急HPの回復、危険イベントの予告開始と本番開始: 即時切替。

`intentSwitchReason`には`minimumCommit`、`hysteresis`、`betterUtility`、`targetUnavailable`、`targetStalled`、`emergency`、`phaseTransition`を残します。

## 回収Utility

回収候補は距離、失効時刻、近傍密度で軽量順位を付けた後、安全候補を最低2件得るまで上限内で順次完全評価します。近い候補が`pathUnreachable`や`pathRisk`で落ちても、その外側を評価します。Navigation Portの経路を6区間に分け、経路上へ到達できる敵と敵弾だけを相対運動で評価します。近傍Pickup密度と経路コリドーは150pxセルの決定論的グリッドから検索します。

地上XPは50個以下を制御目標とします。制御は二段階の切替ではなく、20個で0、70個で1となる連続`backlogPressure`を探索距離、候補上限、経路評価幅、XP価値へ反映します。各候補では目的地までの経路上で磁力圏へ入るXP数と値を数え、単一Pickupへの最短距離だけでなく回収コリドーの成果を評価します。50個はゲームルール上の強制削除や厳密上限ではありません。安全包絡を常に先に適用し、危険イベント本番中の一時超過も許容します。

XP:

- `xpValue / xpToNext`の基礎価値。
- 次レベル直前を滑らかに高めるシグモイド差分。
- 近傍に同種Pickupが集まる場合の回収密度。
- ETAと経路リスク。

回復:

- 到着時の予測HPに対する実効回復量。
- ETAが残存時間を超える候補の除外。
- 失効接近度、経路リスク、武器プロファイルの回復優先度。
- 満HPかつ予測ダメージ0なら候補から除外。

満HP時は、移動終点だけでなく現在位置から0.58秒先までの線分が回復磁力圏を横切るかを評価します。安全上必要でない限り、価値0の回復を引き寄せる経路を選びません。

## 危険イベント戦術

`EncounterPhase`を戦術入力として使い、イベントの前後で役割を切り替えます。

| フェーズ | 戦術 |
| --- | --- |
| `pending` | 通常Utility。地上XPに応じた連続圧力で回収範囲と価値を上げる |
| `warning` | 緊急回復を除き、回収後の安全位置まで予告時間内に到達できるXPへ切り替える |
| `active` | XPを意図的な目標から外し、安全包絡内で回避しながら敵数を減らす。緊急回復は許可する |
| `recovery` | 通常Utilityへ戻り、危険中に増えたXPを密集回収する |

予告開始時の交戦から回収、本番開始時の回収から交戦への切替は最低コミットを待たず、`phaseTransition`として記録します。射撃は回収中も継続し、危険中の回避はMovementが戦闘Intentより先に保証します。

## 武器技能

### Pulse

- 敵速度を使う迎撃点。
- 集束中の標的維持と集束追加ダメージを含むTTK。
- 射程内、入場済み、障害物より手前にある敵だけを貫通列として数える。
- 現在の`shotTimer`と弾速を含む短時間撃破Utility。

### Spread

- 標的角、各敵の迎撃角、24方向を候補照準として評価する。
- 全弾の扇形に入る異なる敵数、期待ダメージ、過剰ダメージ、脅威除去を比較する。
- 緊急接触時は群れ評価より対象への確実な命中を優先する。
- Sweep成立に必要な異なる標的数と次射撃短縮をTTKへ反映する。

`aimExpectedDistinctHits`を共通テレメトリへ出し、Pulseの列技能とSpreadの扇形技能を同じ「到達可能な異なる標的数」として比較できます。

## 訪問履歴巡回

`visit-history-v1`は、現在の崩壊Inset内を3 x 3へ分け、障害物と外周余白を除いた到達可能地点の訪問時刻をAgent内に保持します。固定された四隅を順番に踏まず、未訪問または最終訪問が古い領域を経路ETAと開放余地で選びます。

- 訪問時計は`pending` / `recovery`だけ進め、`warning` / `active`中は停止する。
- 実際に72pxの領域へ入った場合だけ訪問済みにする。
- 目標が到達不能、崩壊で消失、0.9秒進捗なしの場合は一時保留して別領域を選ぶ。
- 15秒から30秒の領域年齢と危険度から巡回Utilityを上げ、危険度0.2までは低圧、0.55以上では追加値を0にする。
- 連続`backlogPressure`を持つ安全なXP、危険予告中のXP、危険本番、緊急時は巡回より先に確定する。
- 安全な直接経路がない場合だけ、古さとセル内XP密度を合わせた領域を候補にする。
- 領域経路は通常1秒キャッシュし、崩壊・障害物変更と目標到達時だけ即時更新する。
- Movementの安全包絡は変更せず、同じ安全帯内でだけ巡回目標への進捗を評価する。

現行`periodic-v3`の24シード x 2武器、最大450秒Baselineでは、1秒以上の巡回が0 / 48、巡回Intentが0.069%でした。このため既存`patrol`の方向だけを置換する案を停止し、上記の期限付き候補へ改訂しました。

初回候補の24シード比較は訪問率を1.7ポイント、生存を3.7秒改善しましたが、平均地上XP、非activeの50超過率、敵弾被ダメージ、満HP回復が悪化しました。第二次監査対応後の開発3シード x 2武器、450秒比較でも、`visit-history-v1`は`periodic-v3`に対して平均地上XPが37.59個から38.04個へ増え、平穏時50超率の改善も14.66%から14.43%に留まりました。既定は`periodic-v3`を維持し、訪問記憶は将来の明示的なXPコリドータスクの特徴量として残します。

## 診断と性能

Decisionとブラウザのデバッグスナップショットは次を分離して公開します。

- `intentMode` / `intentTargetId`。
- `executedMode` / `overrideReason`。
- `riskScore` / `minimumTtc` / `predictedDamage`。
- `intentUtility` / `intentPathEta` / `intentCommitSeconds`。
- `aimExpectedDistinctHits`。
- `motionDisposition`、実変位、目標進捗、waypoint、選択速度と目的速度、安全候補数。
- Pickupの入力数、距離内、軽量候補、経路評価、到達可能、TTL有効、安全候補、除外理由。
- 巡回戦略、目標領域、遷移理由、30秒 / 120秒窓の訪問率、最古領域年齢。
- 地上XPの平均、最大、50個超過フレーム、危険フェーズ別Intent比率。
- XP意図中の停止フレーム、回避上書きなしの停止、最長連続停止時間。

プローブはさらに、0.5秒窓の実変位と目標進捗、未分類停止、移動候補がある状態の安全停止、停滞解除数、経路コリドー回収数、目標切替理由、回収中の回避上書き、回収コミットから取得後1秒までの被弾、無駄回復、回復効率、照準標的数、`targeting / tactics / movement / total`のp95を集計します。性能計測は明示時だけ有効にし、通常の決定論テストへ実時間を混ぜません。

| Pulse | 9730 | 24 | 0 | 6 | 1.12 | 1.43ms |
| Spread | 9678 | 32 | 0 | 16 | 1.66 | 1.39ms |

この1シードの短時間結果は、動作確認と性能予算の確認だけに使います。武器バランスの結論には使いません。

同日のceiling、同シード、30fpsでフェーズ戦術を確認した結果:

## 実行方法

通常の決定論テスト:

```bash
cd phaser
npm test
```

固定シードの詳細プローブ:

```bash
cd phaser
VITE_ARENA_AUTO_PILOT_PROFILE=1 \
VITE_ARENA_AUTO_PILOT_PROFILE_ID=fair \
VITE_ARENA_AUTO_PILOT_DURATION=450 \
VITE_ARENA_AUTO_PILOT_SEEDS=20260711,20260712 \
VITE_ARENA_AUTO_PILOT_PATROL_STRATEGY=visit-history-v1 \
npx vitest run src/simulation/autoPilotProbe.test.ts -t "long fixed-seed"
```

ブラウザでは`?autopilot=pulse`または`?autopilot=spread`でceilingプロファイルを開始し、`O`で手動操作へ切り替えます。巡回候補を観戦する場合は、起動時に`VITE_ARENA_AUTO_PILOT_PATROL_STRATEGY=visit-history-v1`を指定します。ランには`auto-pilot:tactical-observer-v3`と`auto-pilot:patrol:visit-history-v1`が付きます。

## 凍結実験計画

調整用と評価用のシードを混ぜません。

| 集合 | 規模 | 上限 | 用途 |
| --- | ---: | ---: | --- |
| マイクロケース | 手書き | 数秒 | 離反弾、遮蔽、回復、扇形、コミットの回帰 |
| 開発セット | 24シード | 450秒 | 定数調整とアブレーション |
| ホールドアウト | 64シード | 650秒 | 設定凍結後の最終比較 |

PulseとSpread、BaselineとCandidateは同じシードを組にします。打ち切りランは通常の中央値だけで扱わず、Kaplan-Meier、300 / 450 / 600秒生存率、650秒までのRMST、シード単位のペア・ブートストラップ95%区間を使います。

追跡KPI:

- 安全回収機会率と回収ETA。
- 回収コミットから取得後1秒までの被ダメージ率。
- 回復効率と満HP回復取得。
- 自発的な目標切替回数。
- 回避可能なハード危険違反。
- 650秒RMSTの非劣性。
- 同一入力の出力一致。
- 後半状態のAI判断p95 1.5ms。

## ライブラリ判断

- rot.js: 継続採用。現在の小規模2D Dijkstraに適合する。
- Velocity Obstacles: 相対速度とTTCの数学だけを自前実装へ採用する。
- ORCA / RVO: 敵が回避責任を負わないため不採用。
- Behavior Tree / GOAP: 中心課題が行動列ではなく連続危険と価値の比較なので不採用。
- Yuka: World同期層と独自Utilityが残るため不採用。
- navcat / recast-navigation-js: 現在の固定矩形4障害物には過大。複雑な地形導入時に再評価する。
