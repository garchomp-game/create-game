## この文書の位置づけ

このページは、2026-07-20に共有された外部助言を失わず、設計要件へ変換する前の会話メモとして残すものです。助言はそのままゲームルールへ採用せず、RC6 baseline固定後のWork再レビュー、外部プレイテスト、Issueごとの採否を通して判断します。

公開ドキュメントへ個人名とアカウント名を残さないため、発言者は「ゲーム業界経験者」と表記します。本文は識別情報を除いて原文を保持しています。

## 共有された助言

> ゲームバランスには設計上の「鉄板」が存在します。
>
> 目指す状態として下記3点を設定する場合：
>
> - シンプルなゲーム性（分かりやすさ）
> - ついつい次のステージを挑戦したくなる（継続性）
> - やめられない中毒性（没入感）
>
> バランス設計のポイント：
>
> 1. 緊張と緩和を繰り返すことで、リテンションさせる（感情の波を意図的に設計し、体験させる）
> 2. ボスは単に「硬い＝HPが高い」のではなく、攻撃パターンや攻撃密度などの戦略性で「強さ」調整する
> 3. 初見で攻略させない場合、何等かの介入要素を持たせる（プレイヤーが強くなるためのオプションが、強敵へのメタ要素として機能）
> 4. 「あとちょっとのところで敗北させる」＝次はできそう！と感じさせる。「〇回以上失敗して、一度プレイを止めて、戻ってきた、というユーザーには難易度をわずかに下げる」＝継続すれば成功に至る！を体験させる
>
> 無双感でアドレナリンを出させ、停滞感でクリアへの渇望感を高める、の繰り返しをいかにできるか。
>
> そして、挑戦の末クリアできたことを「自分がうまくやった」と錯覚させることが重要です。
>
> ステージクリア型の、シューティングゲームやパズルゲームに必ず入っているロジックです。

## Arena Coreへの一次解釈

| 助言 | 現在の接点 | 採否前に決めること |
| --- | --- | --- |
| 緊張と緩和 | Encounter Director、Act、危険予告、強化停止、回復経路 | EndlessとExpeditionで強度曲線を分け、ピーク後の立て直し時間を数値化する |
| HP以外のボス強度 | `targeted-salvo`、`escort-pincer`、`command-pulse`、遮蔽 | HPを上げる前に、位置変更、射線、標的優先、反撃時間でphaseを設計する |
| 強敵への介入 | 通常強化、武器覚醒、EX、契約、将来の武器教義 | 必須対策が重み付きランダムだけに依存しない準備選択を作るか判断する |
| 惜敗と再挑戦 | 到達Act、ボス残HP、攻撃別被弾、履歴、リザルト | 「あと少し」を偽装せず、前回との差と次に変えられる行動を表示する |
| 無双感と停滞感 | 強化直後の火力上昇、危険event、ボスphase | 無双区間を恒常化せず、短いpower spikeと次の判断要求を交互に置く |
| 自分の上達として理解する | 技能shadow ledger、武器別成果、次ラン目標 | 隠れ補正ではなく、成功した判断を本人へ返して有能感を支える |

## 公平性の境界

「失敗回数や復帰後に難度を下げる」案は、Campaignの継続支援としては検討できますが、Endlessとランキング対象ランへ隠れて適用しません。

- Endless、fixed seed、ランキング対象は同じrulesetと入力条件で固定する。
- Campaign支援を導入する場合は、明示的な支援状態、rulesetまたはdifficulty ID、ランキング対象外理由を記録する。
- まずは失敗理由、ボス残HP、到達phase、次の一手を表示し、数値補正なしで再挑戦が生まれるか検証する。
- 支援を使わない選択肢を残し、プレイヤーに隠れて敵HPや攻撃密度を変えない。
- 「錯覚させる」は操作と判断の成果を本人が認識できる状態へ読み替え、誤認を意図的に作らない。
