## 目的

[#79 Pulse / Spreadの武器教義ブランチ](https://github.com/garchomp-game/create-game/issues/79)を、単純な火力強化や新しい抽選候補の追加にしないため、現行挙動と必要な比較を固定します。

このページは実装前の候補比較です。名称、倍率、発動回数、取得UIは未採用です。[#83](https://github.com/garchomp-game/create-game/issues/83)、[#76](https://github.com/garchomp-game/create-game/issues/76)、[#77](https://github.com/garchomp-game/create-game/issues/77)の判断後に1 candidateずつ事前登録します。

## 現行の武器差

| 軸 | Pulse | Spread |
| --- | --- | --- |
| 基礎射撃 | 0.12秒、1発、damage 1.16、speed 520 | 0.33秒、3発、damage 1.00、speed 480 |
| 通常固有 | 集束共鳴2段階 | 分裂射撃2段階、最大5発 |
| 最終強化 | 障害物・外周で1回反射 | 1斉射3標的で次弾を加速 |
| 主判断 | 優先標的への照準維持、貫通順、反射角 | 複数標的が扇形へ入る距離と位置 |
| 既存成果 | 集束追加damage、貫通列、反射後hit / kill | 異なる命中敵数、掃射trigger / consume |

共有強化は連射、移動、最大HP、弾速、貫通です。弾速だけPulseへ高い武器別倍率があります。通常ビルドは両武器とも関連25ランクをすべて取得してからEXへ移ります。

現状でも射撃形状は同一化していません。v0.8の課題は新しい形を足すことではなく、熟練者が現在の役割を意図的に深める選択を作ることです。

## 現行計測で分かること

### 共通

- shot、projectile、hit、kill。
- hitした斉射数、hit斉射あたりの異なる敵数、敵種別hit / kill。
- 生存、score、damage、回復、総移動距離、ビルド完成時刻。

### Pulse

- 集束強化hit、対象継続分と貫通列分の追加damage、最大stack。
- 障害物 / 外周 / 外周面別の反射回数、反射後hit / kill、異なる追加標的数。

### Spread

- 掃射trigger / consume、1斉射の最大異なる標的数。

これらはrun exportへ残り、固定probeでも一部を要約できます。

## 計測不足

| 不足 | 影響 |
| --- | --- |
| 弾道と敵中心の正規化距離 | Pulseの精密照準と単なるhit率を分けられない |
| 集束維持時間、対象切替、切断理由 | 集束が狙って維持されたか、偶然続いたか分からない |
| 到達可能な貫通列と実利用数 | 敵が並ばなかったrunと、列を逃したrunを分けられない |
| Spreadの異なる標的数分布 | 最大値だけでは、普段何体を捉えたか分からない |
| 掃射までの時間、連続成功、未消費終了 | 掃射循環が戦術になったか分からない |
| 発射時の敵扇形占有と距離帯 | Spreadが群れへ位置を合わせたか分からない |
| 照準、player位置、優先標的の短時間sample | 取得前後で操作が変わったことを説明できない |

詳細sampleは開発run exportへ上限付きで残し、通常のRunRecordへ毎frame保存しません。技能要約は#77のshadow ledgerへ接続します。

## candidate比較

### Pulse

| candidate | プレイ行動 | 報酬の方向 | リスク | 判断 |
| --- | --- | --- | --- | --- |
| P-A Core Lock | 敵の中心へ近い弾道を連続で通す | 集束の維持または次hitの集束効率 | 小型Fastが過度に不利、当たり判定が不可視だと不公平 | 第一候補 |
| P-B Rebound Route | 障害物・外周の反射後に別標的へ当てる | 反射後の処理窓または再反射準備 | 既存反響回路と重複し、外周連射だけで自動発動し得る | 保留 |

P-Aは「中心命中ならbonus、外側なら現状damage」を原則とします。外側hitを弱体化しません。中心度は色だけでなく、小さなcore markと固有hit音で説明します。

最初は中心度をtelemetryだけで測り、人間とobserver-fairの分布を確認します。閾値を実ラン結果に合わせて有利に選ばず、候補値をfixtureで先に固定します。

### Spread

| candidate | プレイ行動 | 報酬の方向 | リスク | 判断 |
| --- | --- | --- | --- | --- |
| S-A Sweep Chain | 連続する斉射で異なる敵を複数捉える | 短いtempo windowを維持 | 現行掃射の単純上位化、密集時の自動発動 | 第一候補 |
| S-B Close Break | 近距離の扇形へ複数敵を入れて撃つ | 次の退避または制圧機会 | 接触damageとのtradeが強く、単体damage化するとPulseへ接近 | 保留 |

S-Aは1回の偶然な3体hitではなく、次の斉射でも群れを扇形へ収める行動を評価します。damage倍率ではなく、短い射撃tempoまたは掃射準備の維持を候補にします。

S-Bを将来試す場合も、近距離単体damage bonusにはしません。退避経路、軽い押し戻し、扇形の一時拡張など、面制圧のまま深めます。

## 推奨する最初の順序

二つの武器を同時に変更しません。

1. #77で中心度、集束切断、Spread標的数分布、掃射連続成功を計測する。
2. 完成済み通常ビルドの固定fixtureで、candidateなしの操作指標を凍結する。
3. P-Aだけをfeature flagで有効にし、Pulse controlと比較する。
4. P-Aを採用、再調整、棄却のいずれかへ固定する。
5. S-Aだけを別flagで有効にし、Spread controlと比較する。
6. 両方を採用した場合だけ、統合rulesetで武器間の非劣性を確認する。

最初の実験ではproductionの選択UIを追加せず、run configでcandidateを固定します。採用後の教義選択は通常25ランク完成後またはEX入口へ置き、通常強化の重み付き抽選へ混ぜません。

## 通常強化の候補運を分ける

通常強化は現在、利用可能候補から毎回重み付きで3件を選びます。最大済み候補は消え、最終強化は解放時に優先提示されますが、EXのような一巡保証はありません。

そのため、連射や移動などの基礎強化が長く出ないrunは成立します。これはビルド多様性でもありますが、教義candidateの成果と混ぜると比較を汚します。

- micro fixtureでは通常ビルドを直接完成状態へし、候補運を除外する。
- fixed runではoffer列、初回出現level、最大未提示gapを武器別に残す。
- productionへ一巡保証や最低提示回数を入れるかは、別の進行判断として扱う。
- 特定の`rapidFire`だけを必須化せず、攻撃・機動・生存のどの最低保証が必要かを先に決める。

## fixture

### Pulse

1. 大型静止敵への中心 / 中間 / 外縁hitを同じdamageのcontrolで分類する。
2. 移動Fastへの中心判定がframe rateで変わらない。
3. 同じ敵への継続、対象切替、期限切れ、撃破で切断理由を分ける。
4. 2体から4体の直線配置で到達可能列と実hit順を固定する。
5. 反射後hitはP-Aの中心連続へ混ぜない。

### Spread

1. 0体から5体を扇形内へ置き、異なる標的数分布を固定する。
2. 同じ敵への複数弾は1標的として数える。
3. 連続成功、失敗、時間切れ、未消費終了を分ける。
4. 近距離と遠距離で扇形占有を比較する。
5. 30、60、120、144Hzでtrigger / consume列を一致させる。

### 共通

- candidate無効時にRC6のevent / world hashが一致する。
- Pulse / Spreadを同一seed、同一難易度、同一完成build時刻で比較する。
- observer-fairでは移動安全係数を共通化し、武器専用照準だけを許す。
- 手動では取得前後の照準、位置、標的選択を短い動画または観測票で残す。

## 採否基準

- candidateを一文の操作で説明できる。
- 取得前後で照準、位置、標的優先の少なくとも1つが変わる。
- 成功が専用telemetryで増え、総scoreだけに依存しない。
- 失敗時も既存武器の基礎性能を下げない。
- observerだけが完全情報で成功する仕組みにしない。
- 片方のcandidateが全敵・全距離で既存選択の上位互換にならない。
- candidateなしへ設定差分だけで戻せる。

## 対象外

- Pulseへの分裂弾復活。
- Spreadの一律damage / 弾数nerf。
- 通常強化表、EXサイクル、敵HPの同時変更。
- 3つ目の製品武器。
- 教義成果のランキング直接加点。
- 観戦AIの技能上限を人間の合格条件にすること。
