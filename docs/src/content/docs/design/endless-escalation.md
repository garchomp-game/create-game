## 目的

通常ビルド完成後も判断と成長を止めず、同時に、どれだけ上達しても最終的にはランが終わるエンドレスモードを作ります。

単純に敵数と移動速度を増やし続けると、画面の可読性と性能が先に破綻します。そのため、画面密度に関わる値には上限を置き、敵の耐久・攻撃力、反復イベント、回復圧力、戦場面積で長期的な危険を増やします。

## 現在の流れ

1. 通常XPで既存強化と最終強化を取得する。
2. 通常候補がなくなると、`buildCompletedAt`を記録してEX進行へ移る。
3. EXレベルごとに、現在のEXサイクルで未取得の限界強化から選ぶ。
4. 240秒以降は脅威ティアにより敵の能力が上がり続ける。
5. 3種類の危険イベントをシャッフルバッグで反復する。
6. 600秒から安全領域が段階的に狭まり、領域外で継続ダメージを受ける。

## EXレベル

通常強化の最大ランク表示とランクは維持し、EX強化を別のIDとランクで記録します。

| 強化 | 1ランクの効果 | 長期上限 |
| --- | --- | --- |
| 限界出力 | 弾ダメージ+8% | なし |
| 過給サイクル | 追加連射速度+10% | 5段階 / +50% |
| 超過駆動 | 追加移動速度+6% | 5段階 / +30% |
| 増設コア | 最大HP+8 | なし |

EX必要XPは180から1.12倍で増え、900で止まります。連射と移動は操作性と描画負荷のため5段階の上限を持ち、上限へ達した候補は次のサイクル補充から外します。限界出力と増設コアは残り続けるため、提示された候補が無効果にはなりません。

### EXサイクル

EXは通常強化のような独立抽選ではなく、取得順を考える小さなデッキとして扱います。

1. サイクル開始時に、上限未到達のEX候補を未取得一覧へ補充する。
2. 未取得一覧から最大3件を、強化用乱数列で決定論的に提示する。
3. 選んだ候補は現在サイクルの一覧から除外し、同じサイクル中には再登場させない。
4. 残り1件になった場合は、その候補を自動取得して選択画面による中断を省く。
5. 一覧を使い切った次のEXレベルで、新しいサイクルを開始する。

これにより、同じ強化を連続で引き続ける偏りを抑えつつ、どの候補を先に取るかという判断を残します。ランダムな新能力そのものはEXへ混ぜず、将来の変異、レリック、イベント、ボス報酬として別の希少獲得系へ分離します。

プレイヤー側は線形に成長し、敵の耐久と攻撃力は脅威ティアごとに乗算で成長します。したがって、EXレベルは延命とビルド判断を提供しますが、敵成長を恒久的には追い越しません。

## 脅威ティア

脅威ティアは240秒で1になり、以後45秒ごとに1増えます。ティア自体に上限はありません。

| 値 | ティアごとの変化 | 上限 |
| --- | ---: | ---: |
| 敵HP | x1.12 | なし |
| 接触・敵弾ダメージ | x1.10 | なし |
| 敵スコア | x1.08 | なし |
| 遠距離弾速度 | x1.04 | x1.80 |
| 遠距離攻撃頻度 | x1.05 | x2.40 |
| 回復出現倍率 | x0.96 | x0.35を下限 |

90秒以降の出現圧力も60秒ごとに更新します。出現間隔は0.32秒、敵速度倍率は2.15、同時敵数は96、出現予算は8を上限とします。これは難易度の停止ではなく、可読性と処理負荷を守るための密度上限です。

## 危険イベントデッキ

現在は次の3種類を、同じシードなら同じ順になるシャッフルバッグで反復します。バッグを使い切るまで同じイベントは重複せず、補充時も直前と同じイベントを先頭にしません。

| イベント | 主な敵 | 要求する判断 |
| --- | --- | --- |
| 射撃体集中 | 遠距離敵 | 射線管理と優先撃破 |
| 高速群襲来 | 高速敵と追跡敵 | 進路変更と包囲回避 |
| 重装包囲 | 重量敵 | 射線確保と処理順 |

各イベントは予告、活動、回復の3区間を持ちます。イベント間隔は脅威ティアに応じて短くなりますが、26秒を下回りません。初回イベント後の次予告は、240秒の契約画面と重ならないよう最低8秒の余白を取ります。

この仕組みはv0.7の構造化出現カードへ進む前段です。現状は敵の重み、出現間隔、予算だけを差し替え、出現方向や隊形はまだ通常抽選を使います。

## アリーナ崩壊

600秒から45秒ごとに安全領域を各辺18pxずつ狭めます。領域外では0.5秒ごとにダメージを受け、段階ごとにダメージが1.12倍になります。

収縮は次の役割を持ちます。

- 外周を回り続けるだけの安定解を崩す。
- 障害物、敵弾、危険イベントとの組み合わせで位置判断を作る。
- 最終的に安全領域を消し、任意の瞬間死タイマーを使わずランを終わらせる。

赤い領域と安全境界を常時描画し、縮小8秒前からHUDで予告します。崩壊ダメージは接触・敵弾と別の死因として記録します。

## 記録と比較

ラン記録と開発ログへ次を追加します。

- EXレベル、EXサイクル、EX強化ランク、選択履歴、自動取得の有無。
- 終了時の脅威ティアと崩壊段階。
- 危険イベント別の開始回数と完了回数。
- 崩壊開始時刻、最高段階、崩壊ダメージ。
- 接触、敵弾、崩壊を分けた被ダメージ。

ルール変更前の記録とは競技条件が異なるため、`phaser-v0.6.2-navigation-extra-cycle`を別のランキング比較単位として使います。開発サーバーの`logs/runs/summary.tsv`と`summary.csv`にも同じ主要値を出力します。

## 手動評価で確認すること

- 通常ビルド完成から最初のEX選択までが長すぎないか。
- EX候補の優先順位が状況によって変わり、同一サイクル内の重複抑制が単調さを減らすか。
- 最後の1候補の自動取得が理解でき、取りこぼした印象を与えないか。
- 3イベントを予告名だけで識別でき、対応を変えられるか。
- 600秒以降の収縮が唐突ではなく、回避可能な失敗として感じられるか。
- 敵HP上昇が緊張感より先に単調な硬さへ変わらないか。
- 熟練者のランが終わる一方、通常プレイの5分前後を不必要に難しくしていないか。

自動試験は有限終了、個体数上限、再現性、記録整合性を保証します。面白さ、納得感、適切な終了時刻は複数の手動ランで調整します。
