## アイデアの扱い

| 区分 | 意味 |
| --- | --- |
| 採用方針 | 次フェーズの前提として進める |
| 検証候補 | 小さな試作とプレイテストで採否を決める |
| 後回し | 価値はあるが、依存関係やコンテンツ量が揃うまで作らない |

## 目指す体験

> 狙い、位置取り、地形利用の上達と、毎回異なるビルド判断が噛み合う、短時間で再挑戦したくなるアリーナサバイバル。

継続性は、次の掛け合わせで作ります。

`操作の上達 x ビルドの違い x 読める危機 x 挑戦目標`

## 現状診断

### 成立している部分

- 移動、照準、射撃、回避を同時に扱う上達余地がある。
- 敵4種、敵弾、障害物、回復ピックアップによって最低限の優先順位判断が生まれている。
- 1分未満から10分超までラン結果に幅があり、操作への慣れがスコアへ反映されている。
- シード、ラン出力、自動ログ、バランス回帰テストがあり、変更を比較できる。
- 90秒以降の持久圧力により、長時間ランの敵密度と速度は段階的に上がる。

### 面白さを止めている部分

1. 強化の多くが数値上昇で、武器の使い方そのものは変わりにくい。
2. `pulse` 以外の武器設定はあるが、通常プレイで開始武器として選べない。
3. 敵4種の行動は実質的に追跡系と遠距離系の2系統で、出現構成も重み付き抽選だけである。
4. 90秒で全敵種が揃った後は、同じ構成の数値圧が続く。
5. XP必要量が指数的に増えるため、後半ほど次の強化選択までが急激に長くなる。
6. スコアは主に撃破点の累積で、危険を取る判断や特殊条件の達成を十分に表していない。
7. 出現と強化抽選が同じ乱数列を共有しており、新しい抽選を足すと同じシードの別システムまで変化する。

### プレイログからの根拠

代表的な高スコアランは、51282点、約14.7分、レベル20、強化19回、取得XP 4543でした。

現行式ではレベル20から21に1792 XPが必要です。全強化ランク24へ到達するレベル25までの累積XPは約24088となります。レベル20が上限なのではなく、実プレイ上は選択間隔が伸び、成長が止まったように見えることが問題です。

優先課題は敵を一律に強くすることではありません。

- 30秒から90秒ごとに意味のある判断を戻す。
- 数値ではなく、狙い方や位置取りが変わる強化を作る。
- 同じ敵を増やすだけではない戦闘展開を作る。
- 長時間生存時に、追加能力だけでなく危険と報酬を選ばせる。

## 設計の柱

### 1. 上達でき、読み取れる戦闘

- 色だけで敵種や危険を区別しない。
- 大ダメージ、突進、範囲攻撃には予兆を付ける。
- 防御行動は緊急回避に使えるが、位置取りを無効化する連打手段にはしない。
- 障害物を射線、誘導、跳弾の判断に使えるようにする。

### 2. 説明できるビルド

- 開始武器で最初の方針を変える。
- 90秒以内に、挙動を変える重要な選択を提示する。
- 共通強化と武器固有強化を分ける。
- 最大ランク到達を、分岐や最終強化へつなげる。
- 隠し組み合わせを前提にせず、条件と効果を画面で予告する。

### 3. 段階的に変わる戦闘展開

- 群れ、挟撃、護衛、遠距離陣形などの出現パターンを作る。
- 同じ展開を連続させず、危機の後に短い回収時間を置く。
- エリート、危険度選択、ボスをランの節目として使う。
- エンドレス後半では、危険とスコア倍率を同時に選ばせる。

### 4. 公平で再現可能なランダム性

- 出現、強化、ドロップ、戦闘展開、地形差分の乱数列を分ける。
- 重要な抽選結果をシードとイベント履歴から再現できるようにする。
- 障害物は完全自動生成から始めず、検証済み配置から選ぶ。
- 敵出現には最低距離、侵入予告、逃げ道の制約を持たせる。

### 5. 作業化しないやりこみ

- 解放対象は武器、ステージ、チャレンジ、特殊ルール、外観を優先する。
- 武器熟練度はプレイスタイルの学習と記録に使う。
- 恒久攻撃力や恒久HPは当面採用しない。
- 外部アカウントは、ローカル保存で必要性を確認してから判断する。

## 機能の優先評価

| アイデア | 期待する価値 | 費用とリスク | 判断 |
| --- | --- | --- | --- |
| XP曲線と選択間隔の再設計 | 選択停滞を直接解消する | 全体バランスへ影響する | v0.6最優先 |
| 3種類の開始武器 | ラン開始時から差が出る | 武器別調整が必要 | v0.6最優先 |
| 武器最終強化 | ビルドの個性を作る | 演出と効果合成が必要 | v0.6で試作 |
| 危険度契約 | エンドレス後半へ判断を戻す | スコア比較が複雑になる | v0.6で試作 |
| 戦闘展開カード | 既存の敵から展開差を作れる | 制御役と安全規則が必要 | v0.7最優先 |
| 第2ステージ | 外的な変化が大きい | 地形、ルール、UI、調整が必要 | v0.7後半 |
| 最初のボス | クリア型ランの山場になる | 予兆と専用検証が必要 | v0.7で試作 |
| ダッシュ | 操作ミス救済と上達余地になる | 無敵と衝突判定を壊しやすい | 操作試作 |
| BGMと演出 | 手触りと完成度を上げる | 素材権利と音量調整が必要 | v0.5で限定導入 |
| アイテム大量追加 | 現段階では判断増加が小さい | 効果合成が広がる | 後回し |
| 外部アカウント | 現段階の価値が小さい | 開発と運用負担が大きい | v1.0後に再評価 |

## モード構成

| モード | 役割 | 目安 |
| --- | --- | --- |
| エンドレス | スコア挑戦とビルド実験 | 時間上限なし |
| エクスペディション | 明確な終点を持つステージ攻略 | 8分から10分 |
| チャレンジ | 固定条件で特定技能を試す | 3分から5分 |

詳細は [モードと戦闘展開](../encounters-and-modes/) を参照してください。

## v1.0の最小コンテンツ量

| コンテンツ | 目標 |
| --- | ---: |
| モード | エンドレス、エクスペディション、チャレンジ |
| 開始武器 | 3 |
| 武器最終強化 | 3 |
| ステージ | 2 |
| 通常敵の役割 | 5 |
| エリート特性 | 2から3 |
| ボス | 完成度を確認できる1体 |
| 戦闘展開カード | 5から7 |
| 危険度契約または特殊ルール | 3から4 |
| チャレンジ | 5から6 |

この量で組み合わせの面白さが出ない場合は、コンテンツ追加ではなくシステム設計を見直します。

## v1.0まで扱わないもの

- 外部ログイン、クラウド保存、オンラインランキング。
- 日替わりチャレンジのバックエンド。
- 完全自動生成ステージ。
- 恒久ステータス上昇を中心とした周回。
- 大量の希少度階層と装備収集。
- 対人戦、協力プレイ、物語キャンペーン。

## 判断の関門

1. v0.5でラン記録とリザルトを信頼できる状態にする。
2. 3武器と成長間隔の見直しで、同じアリーナでもラン差が生まれるか確認する。
3. 戦闘展開カードで、新しい敵を増やさず展開差を作れるか確認する。
4. ステージとボスで、8分から10分のクリア型ランが成立するか確認する。
5. チャレンジと熟練度が、強制作業ではなく再挑戦理由になるか確認する。
6. ローカルで継続性を確認した後にだけ、アカウント同期を再検討する。

実装順とチケットは [中長期作業計画](../../project-management/gameplay-expansion-plan/) を正本とします。
