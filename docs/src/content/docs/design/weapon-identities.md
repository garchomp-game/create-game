## この設計の目的

開始武器を選んだ時点で、狙う対象、立つ場所、強化後の戦い方が変わる状態を作ります。

武器差を発射間隔や弾数だけに置くと、強化後に同じ射撃形状へ収束し、最終的には秒間火力の高い方が上位互換になります。共有強化は基礎能力を支えてもよい一方、別武器の初期形態を複製してはいけません。

v0.6.3では次を設計原則として採用します。

1. 武器固有強化は、その武器の初期操作を深める。
2. Pulseは単線の継続照準、Spreadは広角の複数標的を最後まで維持する。
3. 雑魚と高耐久標的を分け、両武器に異なる得意対象を作る。
4. 最終強化は取得前後で狙い方または地形利用を変える。
5. 強さは総合勝率だけでなく、役割別の成果で比較する。

## v0.6.2までの問題

変更前の分裂射撃は、すべての開始武器へ弾を1発ずつ加えていました。

| 武器 | 初期弾数 | 最大弾数 | 基礎射角 | 基礎間隔 | 最大時の基礎弾数/秒 |
| --- | ---: | ---: | ---: | ---: | ---: |
| Pulse | 1 | 3 | 0.12 rad | 0.16秒 | 18.75 |
| Spread | 3 | 5 | 0.52 rad | 0.32秒 | 15.63 |

同じ2ランクでも、Pulseは弾数が3倍、Spreadは約1.67倍です。成長後のPulseはSpreadの初期形態と同じ3方向射撃になり、狭角であること以外の形状差が薄れます。Pulseは弾速、射程、反響回路も持つため、Spreadの価値が広い射角だけに偏ります。

## 武器の役割契約

| 軸 | Pulse | Spread |
| --- | --- | --- |
| 射撃形態 | 単線の連続射撃 | 広角の斉射 |
| 主な判断 | 優先標的へ照準を維持する | 複数標的が射角へ入る位置を取る |
| 得意対象 | Brute、Ranged、縦に並ぶ敵 | Chaser、Fast、横に広がる群れ |
| 地形利用 | 射線、貫通、障害物反射 | 開けた空間、包囲への扇状制圧 |
| 成長方向 | 集中ダメージと直線処理を深める | 弾幕幅と複数命中の循環を深める |
| 明確な弱点 | 横から広がる群れ | 高耐久単体への集中火力 |

どちらも通常ランを完走できる必要はありますが、同じ敵構成に対して同じ操作を要求しません。

## 強化カタログの分離

### 共通強化

- 連射強化: 射撃間隔を短縮する。Pulseは連続火力、Spreadは斉射間隔として体感が異なる。
- 軽快な足取り: 位置取りの余地を増やす。
- 生命コア: 生存余地を増やす。
- 加速弾: 遠距離での命中安定性を上げる。
- 貫通弾: Pulseでは直線処理、Spreadでは扇状の群れ処理を強める。

### Pulse固有強化: 集束共鳴

分裂射撃の代わりに、同じ敵へ直撃を重ねる集中強化を2ランク用意します。

単線化による序盤の処理量低下は、Pulseの基礎射撃間隔を0.16秒から0.12秒へ短縮して補います。最大強化後も弾数ではなく連射密度が伸びるため、Spreadの射撃形状を複製しません。

| 項目 | 仕様 |
| --- | --- |
| 対象 | 跳弾前のPulse直撃 |
| 継続時間 | 最後の直撃から0.9秒 |
| 1スタック | 次回以降の直撃ダメージ +15% |
| ランク1 | 最大2スタック、最大+30% |
| ランク2 | 最大4スタック、最大+60% |
| 解除 | 時間切れ、または対象の消滅 |

スタックは敵ごとに保持します。貫通弾が別の敵へ当たっても、それまでの対象の蓄積を消しません。跳弾後の命中は集束共鳴を蓄積せず、反響回路は範囲補助として役割を分けます。

表示では対象の周囲へ最大4段階の短い共鳴目印を出し、色だけに依存せず蓄積を読めるようにします。

### Pulse最終強化: 反響回路

- 武器カテゴリ7ランクで解放する。
- 障害物で1回跳弾する。
- 跳弾前は集中処理、跳弾後は地形を使った追加処理として扱う。
- 取得時刻、跳弾回数、跳弾後命中、異なる追加命中敵数を記録する。

### Spread固有強化: 分裂射撃

- Spreadだけへ提示する。
- 基礎射撃間隔は0.33秒とする。
- 3発から4発、5発へ増やす。
- 0.52 radの広角を維持し、Pulseの単線射撃へ持ち込まない。
- 1斉射あたりの異なる命中敵数を主要成果とする。

### Spread最終強化: 掃射循環

1斉射で複数標的を捉えると、次の斉射が早まる最終強化です。

| 項目 | 仕様 |
| --- | --- |
| 解放 | 武器カテゴリ7ランク |
| 条件 | 同じ斉射が異なる敵3体へ命中 |
| 報酬 | 掃射チャージを1つ得る |
| 消費 | 次のSpread射撃時 |
| 効果 | 成立時点から次弾までの残り待ち時間を70%にする |
| 上限 | チャージは1つ。1斉射につき1回だけ発動 |

自動射撃中でも次の1回へ確実に適用し、命中時点で射撃待ち時間が残っていなくても報酬を失いません。HUDと取得フィードバックで準備・消費を示します。

## 通常ビルド完成条件

PulseとSpreadは、関連する通常強化の総ランクを同じにします。

| 区分 | Pulse | Spread |
| --- | ---: | ---: |
| 共通5強化 | 22ランク | 22ランク |
| 武器固有強化 | 集束共鳴 2 | 分裂射撃 2 |
| 最終強化 | 反響回路 1 | 掃射循環 1 |
| 合計 | 25 | 25 |

「完成ラン」はゲームクリアではなく、関連する通常25ランクを取得してEX進行へ入ったランを指します。文書とUIでは「通常ビルド完成到達ラン」と表記します。

## 敵耐久との対応

序盤の撃破感とXP速度を変えず、240秒以降に敵役割の差を広げます。基礎HPは変更しません。

| 敵 | 基礎HP | 45秒ごとのHP倍率 | 役割 |
| --- | ---: | ---: | --- |
| Chaser | 1 | 1.12 | 標準雑魚 |
| Fast | 1 | 1.10 | 柔らかい高速脅威 |
| Ranged | 2 | 1.15 | 優先して処理する中耐久標的 |
| Brute | 3 | 1.17 | 照準を維持する高耐久標的 |

600秒付近では現行値と比べ、ChaserとFastはおおむね同等、Rangedは約5から7、Bruteは約8から11HPになります。全敵を一律に硬くせず、Pulseの集中処理とSpreadの群れ処理が同じランに共存する構成を作ります。

HP増加で撃破数とXPが落ちるため、通常ビルド完成時刻、画面内敵数、接触被害を同時に監視します。難易度を上げることと、敵を倒す手応えを増やすことを混同しません。

## 高解像度の選択UI

PhaserのTextやSVGアセットはCanvasへ描画された時点でラスタライズされます。960 x 540のCanvasを拡大するだけでは、元がSVGでも動的な日本語文字はぼやけます。

武器、通常強化、EX強化、後半契約の選択画面は、Canvas上へ重ねるDOM/CSSレイヤーへ移します。

- ブラウザのフォント描画を使い、表示倍率とDPIごとに再描画する。
- 日本語フォントは`Noto Sans CJK JP`、`Noto Sans JP`、`Yu Gothic`、`Meiryo`、sans-serifの順で指定する。
- ボタンは実際の`button`要素とし、ポインター、Tab、Enter、数字キーへ対応する。
- デスクトップは最大3列、縦長画面は縦並びにする。
- Canvasの論理座標へ依存せず、狭い画面でも文字を読める大きさに保つ。
- 画面比較はCanvas単体ではなく、ゲーム親要素を撮影する。

ゲーム中のHUDと弾・敵表示はCanvasへ残します。DOM化は選択とメニューの可読性を優先し、シミュレーション規則から分離します。

## 計測契約

Pulse:

- 集束共鳴の強化命中数、追加ダメージ、最大スタック。
- 集束共鳴中の敵種別撃破。
- 反響回路の発動、跳弾後命中、異なる追加命中敵数。

Spread:

- 斉射あたりの異なる命中敵数。
- 掃射循環の成立回数、消費回数、成立までの平均時間。
- 敵種別命中と撃破。

共通:

- 毎分撃破、毎分スコア、生存時間、被ダメージ。
- 通常ビルド完成時刻、EX開始時刻。
- 敵1体あたりの平均命中数と敵種別の命中数。
- 画面内敵数の最大値と上限滞在時間。

## 採否基準

- Spreadの命中斉射あたり異なる敵数がPulseより15%以上高い。
- PulseのBrute / Rangedへの追加ダメージが記録され、集束共鳴ランク2で最大スタックへ到達できる。
- 自動10シード比較で、生存時間、毎分撃破、毎分スコアの武器差がそれぞれ20%以内に収まる。
- 240秒以前の無入力死亡時間と最初の強化時刻を現行許容範囲へ保つ。
- 通常ビルド完成は4分から6分を維持する。
- 通常ビルド完成後の平均命中数は、直近Pulseランの1.89命中/撃破から2.1から2.6を最初の目標とする。
- 960 x 540、844 x 390、390 x 844、DPR 1 / 2で文字切れと選択不能がない。

数値を満たしても、両武器で同じ場所に立ち同じ方向へ撃つことが最適なら採用しません。手動プレイでは、Pulseの優先標的維持とSpreadの群れを射角へ収める移動が実際に生じたかを記録します。

## 実装境界

- アプリ版とルール版をv0.6.3へ更新する。
- 旧ラン記録は削除せず、新しい計測値を既定値付きで追加する。
- ラン途中保存はないため、旧強化ランクを新しいランへ移行しない。
- 開発用Pierceは今回の製品選択へ追加しない。
- 外部フォント配信へ依存せず、OSのフォールバックを使う。
- 武器固有効果、DOM選択UI、敵耐久を個別の機能境界とテストで無効化できるようにする。
