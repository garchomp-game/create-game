## 目的

ラン終了時に「今回はどのようなビルドだったか」を一文で説明できる状態を作ります。

単純な攻撃速度や移動速度の上昇だけでなく、狙う距離、敵の並べ方、障害物の使い方が変わる強化を優先します。

## 現在の実装

- 通常プレイ開始時に`pulse`または`spread`を選ぶ。`pierce`は開発計測だけに残す。
- 共通5強化、武器固有強化、武器別最終強化を、分類付きの同じカタログで合成する。
- 通常XPは`growth: 1.25`、上限120とし、ビルド完成後はEXレベルへ移る。
- EXレベルは必要XP180から始まり、1.04倍で増えて360を上限とする。
- EX強化は通常強化と別ランク・別サイクルで記録し、攻撃力、連射、移動、最大HPを反復強化する。
- 強化の提示、ロック、最大済み、選択時刻、最終強化成果を記録する。
- Pulseの集束・反射とSpreadの複数標的・掃射成果を手動runとrun exportで確認済み。

## 開始武器

既存設定の3武器を将来の開始選択肢として活用します。v0.6の最初の縦切りでは`pulse`と`spread`だけを製品画面へ出し、`pierce`は固定配置と開発計測で役割を確認してから追加します。

| 武器 | 上達が表れる点 | 強み | 弱み |
| --- | --- | --- | --- |
| `pulse` | 継続照準、貫通順、反射角 | 高耐久・優先標的への集中 | 横に広がる群れへの対応が難しい |
| `spread` | 距離管理と複数標的への角度 | 近距離の群れに強い | 遠距離で弾が散る |
| `pierce` | 敵を一直線に並べる位置取り | 高密度への貫通 | 横に広がる敵や単体に弱い |

v0.6最初の完了条件:

- PulseとSpreadで、照準距離、位置取り、標的優先順位の少なくとも1つが変わる。
- 同じシードで武器別ランを再現できる。
- 単純な秒間ダメージ差だけで優劣が決まらない。
- Pierceを通常選択へ出すか、計測結果から判断できる。

## 強化の分類

強化は次の役割へ分類します。

- 武器: 弾の挙動、射撃パターン、命中時効果。
- 機動: 移動速度、ダッシュ、操作補助。
- 生存: 最大HP、シールド、回復効率。
- 補助: 回収範囲、XP効率、ドロップ操作。
- スキル: 固有行動、使用回数、再使用時間。

実装上は、表示、相乗効果判定、計測で同じタグを使います。

## 分岐と最終強化

最初から各武器へ大量の分岐を作らず、1武器につき1つの明確な最終強化を試します。

実装状況:

- `pulse`: 武器系7ランクで反響回路を解放し、障害物または外周で1回反射する。
- `spread`: 武器系7ランクで掃射循環を解放し、1斉射3標的で次弾を加速する。
- `pierce`: 開発計測だけに残し、製品用最終強化は未採用。

要件:

- 必要ランクと効果を強化画面で事前に確認できる。
- 取得後は弾の見た目とリザルトのビルド要約へ反映する。
- 取得前後で狙い方または地形利用が変わる。

通常ビルド完成後に検討する教義candidateは[武器教義の実装前比較](../weapon-doctrine-preflight/)へ分離しています。

## 通常強化の候補偏り

通常強化は、利用可能候補から毎回重み付きで最大3件を抽選します。最大済み候補は除外し、解放済み最終強化は優先提示しますが、EXのような一巡保証はありません。

このため、連射や移動などの基礎強化が長く提示されないrunは成立します。候補運をビルド多様性として残す範囲と、基礎カテゴリを保証する範囲は未決定です。

検証は[#92 通常強化の候補偏りと基礎ビルド保証](https://github.com/garchomp-game/create-game/issues/92)で行います。

- upgrade / category別の初回提示levelと最大未提示gapをfixed seedで測る。
- 特定の`rapidFire`だけを暗黙の必須強化にしない。
- control、カテゴリ最低保証、通常bagのうち1候補だけを比較する。
- 武器教義の比較fixtureでは通常ビルドを完成状態へし、候補運を除外する。
- reroll、候補固定、通常強化の固定順配布は同時に追加しない。

## 強化選択の間隔

具体的なXP式より先に、体験上の目標を固定します。

| 区間 | 目標 |
| --- | --- |
| 最初の強化 | 開始5秒から10秒 |
| 開始2分まで | 15秒から30秒に1回程度 |
| 2分以降 | 30秒から60秒に1回程度 |
| 重要な挙動変化 | 90秒以内に最低1回 |
| ビルドの主軸完成 | 4分から6分 |
| 主軸完成後 | EX強化と脅威上昇を並行し、成長と危険の競争へ移行 |

通常ビルド完成までは、最後の意味ある通常選択から90秒を大きく超えて空かないことを検証します。完成後はEX提示間隔を別指標として記録します。

## 通常ビルド完成後のEX進行

通常強化は最大ランクに達した項目を候補から外し、すべての関連候補がなくなった時点で完成として固定します。通常ランクを巻き戻したり、同じ通常強化を上限越えさせたりはしません。

EX進行では次を守ります。

- 通常レベルとEXレベルを分けて表示・記録する。
- EX候補はランク上限のない限界出力・増設コアと、5段階の過給サイクル・超過駆動から選ぶ。
- 取得済み候補は同じEXサイクル中に再提示せず、残り1候補は自動取得する。
- サイクル完了後は、上限未到達候補だけを補充して次の取得順を決める。
- 安全上限へ達した候補は抽選から外し、無効果の選択を出さない。
- プレイヤー成長は線形、敵の脅威成長は乗算とし、永久に追い越せる状態を作らない。
- 最大HPを取得したときは増加分だけ現在HPも回復する。
- ランダムな新能力獲得はEXの恒常候補へ混ぜず、将来の変異・レリック・ボス報酬へ分離する。

具体値と後半終了条件は[エンドレス後半の継続圧力](../endless-escalation/)を参照してください。

## エンドレス後半の危険度契約

240秒では、危険と報酬を同時に上げる契約を1回提示します。通常ビルド完成後の成長はEX進行が担当し、契約はラン全体のルール選択として分離します。

| 契約例 | 危険 | 報酬 |
| --- | --- | --- |
| 交差射撃 | 遠距離敵の戦闘展開が増える | スコア倍率上昇 |
| 突撃群 | 高速敵や突進敵が増える | エリート報酬上昇 |
| 物資不足 | 回復出現率が下がる | スコア倍率上昇 |
| 過負荷 | 敵速度と出現予算が上がる | スコアと熟練度上昇 |

最初の危険イベントは135秒から165秒へ配置し、その後は射撃体集中、高速群襲来、重装包囲を反復します。240秒では標準維持か、敵速度1.12倍・スコア1.3倍の過負荷を1回選びます。過負荷は実験ルールとして通常ランキングから除外します。

## スコアとランキング

- 基礎スコアは敵撃破点を維持する。
- 危険度契約に応じて倍率を上げる。
- エクスペディションではクリア加点を使う。
- 生存時間そのものへの連続加点は避ける。
- デバッグ操作、テスト状態、標準外ルールは通常ランキングから除外する。

ランキングの比較単位には最低限、次を含めます。

- `modeId`
- `stageId`
- `rulesetVersion`
- `difficultyId`
- `seedCategory`

連続撃破倍率は武器差を強める可能性があるため、危険度倍率を検証した後の候補とします。

## ランをまたぐ進行

最初のローカル進行は、次の順にします。

1. ラン履歴と自己最高記録。
2. 開始武器の解放。
3. ステージクリアとチャレンジ評価。
4. 武器熟練度と収集状況。
5. 外観、称号、任意の特殊ルール解放。

解放は長時間作業ではなく、遊び方を教える短い目標にします。

例:

- `pulse` で3分生存して `spread` を解放する。
- 1ランで一定数の貫通撃破を達成して `pierce` を解放する。
- エクスペディションをクリアして次のステージを解放する。

## 計測項目

- 強化提示と選択の時刻。
- 最初の強化までの時間。
- 最初の挙動変化までの時間。
- 最終強化までの時間。
- 意味ある選択が空いた最長時間。
- 武器別の生存時間、毎分撃破数、毎分スコア、被ダメージ源。
- 1射あたりの異なる命中敵数、敵種別撃破、移動距離。
- 提示されなかった強化と、提示されたが選ばれなかった強化の区別。
- 危険度契約の提示、選択、到達スコア。
- EXレベルの開始時刻、サイクル、提示回数、選択順、自動取得、各EXランク。
- 終了時の脅威ティア、崩壊段階、崩壊ダメージ。

実装チケットは[v0.6チケット詳細](../../project-management/v06-tickets/)を正本とします。
