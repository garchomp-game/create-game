## 目的

ラン終了時に「今回はどのようなビルドだったか」を一文で説明できる状態を作ります。

単純な攻撃速度や移動速度の上昇だけでなく、狙う距離、敵の並べ方、障害物の使い方が変わる強化を優先します。

## 現在の実装

- 通常プレイ開始時に`pulse`または`spread`を選ぶ。`pierce`は開発計測だけに残す。
- 通常6強化とPulse最終強化「反響回路」を、分類付きの同じカタログで合成する。
- XPは`growth: 1.25`、上限140とし、ビルド完成後は空のレベルアップを出さない。
- 強化の提示、ロック、最大済み、選択時刻、最終強化成果を記録する。
- 手動プレイで武器差と反響回路の地形利用を確認することが残っている。

## 開始武器

既存設定の3武器を将来の開始選択肢として活用します。v0.6の最初の縦切りでは`pulse`と`spread`だけを製品画面へ出し、`pierce`は固定配置と開発計測で役割を確認してから追加します。

| 武器 | 上達が表れる点 | 強み | 弱み |
| --- | --- | --- | --- |
| `pulse` | 継続照準と標的切り替え | 安定して扱える | 特化時の瞬間火力が低い |
| `spread` | 距離管理と複数標的への角度 | 近距離の群れに強い | 遠距離で弾が散る |
| `pierce` | 敵を一直線に並べる位置取り | 高密度への貫通 | 横に広がる敵や単体に弱い |

v0.6最初の完了条件:

- PulseとSpreadで、照準距離、位置取り、標的優先順位の少なくとも1つが変わる。
- 同じシードで武器別ランを再現できる。
- 単純な秒間ダメージ差だけで優劣が決まらない。
- Pierceを通常選択へ出すか、計測結果から判断できる。

## 強化の分類

強化は次の役割へ分類します。

- 武器: 弾の挙動、射撃パターン、命中時効果。
- 機動: 移動速度、ダッシュ、操作補助。
- 生存: 最大HP、シールド、回復効率。
- 補助: 回収範囲、XP効率、ドロップ操作。
- スキル: 固有行動、使用回数、再使用時間。

実装上は、表示、相乗効果判定、計測で同じタグを使います。

## 分岐と最終強化

最初から各武器へ大量の分岐を作らず、1武器につき1つの明確な最終強化を試します。

候補と実装状況:

- `pulse`: 武器系7ランクで反響回路を解放し、障害物で1回跳弾する試作を実装済み。
- `spread`: 拡散幅を絞り、近距離の集中射撃へ変える。
- `pierce`: 貫通後も速度を保ち、敵を並べた見返りを強める。

要件:

- 必要ランクと効果を強化画面で事前に確認できる。
- 取得後は弾の見た目とリザルトのビルド要約へ反映する。
- 取得前後で狙い方または地形利用が変わる。

希少度、再抽選、除外、固定は、強化候補が十分に増えてから扱います。候補が少ない段階では、選択操作を増やしても判断は増えません。

## 強化選択の間隔

具体的なXP式より先に、体験上の目標を固定します。

| 区間 | 目標 |
| --- | --- |
| 最初の強化 | 開始5秒から10秒 |
| 開始2分まで | 15秒から30秒に1回程度 |
| 2分以降 | 30秒から60秒に1回程度 |
| 重要な挙動変化 | 90秒以内に最低1回 |
| ビルドの主軸完成 | 4分から6分 |
| 主軸完成後 | 追加能力より危険度選択へ移行 |

10分ランで、最後の意味ある選択から90秒を大きく超えて空かないことを検証します。

## エンドレス後半の危険度契約

ビルドの主軸が完成した後は、能力を無制限に足すのではなく、危険と報酬を同時に上げる選択を出します。

| 契約例 | 危険 | 報酬 |
| --- | --- | --- |
| 交差射撃 | 遠距離敵の戦闘展開が増える | スコア倍率上昇 |
| 突撃群 | 高速敵や突進敵が増える | エリート報酬上昇 |
| 物資不足 | 回復出現率が下がる | スコア倍率上昇 |
| 過負荷 | 敵速度と出現予算が上がる | スコアと熟練度上昇 |

135秒から165秒へ射撃体集中を配置し、6秒予告、18秒集中、8秒回復で構成しています。240秒では標準維持か、敵速度1.12倍・スコア1.3倍の過負荷を1回選びます。過負荷は実験ルールとして通常ランキングから除外します。

## スコアとランキング

- 基礎スコアは敵撃破点を維持する。
- 危険度契約に応じて倍率を上げる。
- エクスペディションではクリア加点を使う。
- 生存時間そのものへの連続加点は避ける。
- デバッグ操作、テスト状態、標準外ルールは通常ランキングから除外する。

ランキングの比較単位には最低限、次を含めます。

- `modeId`
- `stageId`
- `rulesetVersion`
- `difficultyId`
- `seedCategory`

連続撃破倍率は武器差を強める可能性があるため、危険度倍率を検証した後の候補とします。

## ランをまたぐ進行

最初のローカル進行は、次の順にします。

1. ラン履歴と自己最高記録。
2. 開始武器の解放。
3. ステージクリアとチャレンジ評価。
4. 武器熟練度と収集状況。
5. 外観、称号、任意の特殊ルール解放。

解放は長時間作業ではなく、遊び方を教える短い目標にします。

例:

- `pulse` で3分生存して `spread` を解放する。
- 1ランで一定数の貫通撃破を達成して `pierce` を解放する。
- 最初のエクスペディションをクリアして第2ステージを解放する。

## 計測項目

- 強化提示と選択の時刻。
- 最初の強化までの時間。
- 最初の挙動変化までの時間。
- 最終強化までの時間。
- 意味ある選択が空いた最長時間。
- 武器別の生存時間、毎分撃破数、毎分スコア、被ダメージ源。
- 1射あたりの異なる命中敵数、敵種別撃破、移動距離。
- 提示されなかった強化と、提示されたが選ばれなかった強化の区別。
- 危険度契約の提示、選択、到達スコア。
