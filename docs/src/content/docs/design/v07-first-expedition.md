最終整理日: 2026-07-17

:::note[決定状態]
このページはv0.7の体験要件の正本です。RC2を初回学習ステージとした判断は、手動所感を受けて撤回しました。現行`final-expedition`はStage 1 / 5 / 10の3作戦検証におけるStage 10最終試験です。Stage 1と5、世界観の固有名詞、最終アートは未実装です。Stage 2から4、6から9は3作戦の採否後へ延期します。

RC5は回帰証跡として保持します。production候補は、Encounter時計、Commander期限、記録scope、ruleset、有限回復候補を再検証するRC6です。
:::

## 製品仮説

v0.7では、開始から勝利までの起伏を持つ有限ランを1本完成させます。

> 四方から続く高密度の襲来へ位置取りと標的優先で対応し、ビルドを完成させ、通常増援と広域射撃が続く中で最終指揮艦を撃破する。

この縦切りで検証するのは、構造化した展開が同じ操作基盤へドラマと再挑戦理由を加えられるかです。初心者向け導入は、この基盤を再利用した第1から第4ステージで別途作ります。

## モードの位置づけ

| モード | 目的 | 終了 | v0.7での扱い |
| --- | --- | --- | --- |
| Endless | 防衛線が崩壊するまで戦果と生存時間を伸ばす | 撃墜または安全領域消失 | 現行ルールを回帰基準として維持 |
| Expedition | 有限の戦況を突破して目標を撃破する | ボス撃破で勝利、HP 0で敗北 | 第10ステージ候補を実装 |
| Challenge | 固定条件で技能評価を競う | 条件達成または撃墜 | v0.8以降 |

内部IDはモード`expedition`、ステージ`final-expedition`、ボス`final-command-ship`です。表示名や世界観が変わっても、保存記録の安定IDは変えません。

## プレイヤーへの約束

1. 局面ごとに、位置取り、標的優先、回収、強化のいずれかを考え直す必要がある。
2. 襲来方向とボス攻撃は、被弾する前に形、位置、音の2系統以上で読める。
3. Pulseは精密射撃と優先標的処理、Spreadは扇状の群れ処理という役割を最後まで維持する。
4. 最終ボスでも通常敵が止まらず、雑魚処理とボス攻撃を同時に判断する。
5. 失敗時は、どの局面、敵、攻撃で崩れたかをリザルトとラン記録から説明できる。

## 敵の導入順

手動所感では、小型の黄緑高速体は小ささと速度の両方で対処難度が高く、大型の黄色重装体は当てやすいため低難度でした。RC3はこの差を学習順へ反映します。

| 解禁 | 敵 | 役割 |
| --- | --- | --- |
| 0:00 | 赤の追跡体 | 移動と照準の基準 |
| 1:15 | 大型の黄色重装体 | HP 8の耐久目標。集中火力と分断を要求 |
| 3:00 | 小型の黄緑高速体 | 先読み照準と早期処理を要求 |
| 5:00 | 紫の射撃体 | 射線と敵弾回避を追加 |

色だけで役割を伝えない最終アートは別途必要です。現段階ではサイズ、速度、弾の有無でも区別できます。

## 1ランの構成

イベントを秒数だけでScene分岐せず、ActとEncounter Directorが進行を所有します。

| Act | 初期時間帯 | 役割 | 内容 |
| --- | --- | --- | --- |
| 1 四方警戒 | 0:00から1:15 | 基本操作を確認 | 赤のみ、四方から低圧侵入 |
| 2 第一襲来 | 1:15から3:00 | 耐久目標を分断 | 赤と大型黄、挟撃 |
| 3 反撃 | 3:00から5:00 | 優先標的と高速体へ対応 | HP 500のCommander、小型黄緑 |
| 4 包囲突破 | 5:00から約6:30 | 射撃体を含む複合圧力 | 黄緑と紫、Charger |
| 5 最終決戦 | 約6:30以降 | 全技能を統合 | 通常ウェーブ、追跡ボス、広域射撃、護衛 |

長期化時も通常ウェーブの上昇を止めません。最終ステージなので安定勝利は保証しませんが、即死や不可避攻撃ではなく、読み取れる圧力で詰ませます。

## ステージ契約

`StageDefinition`はアリーナ、障害物、敵プール、遭遇デッキ、難度、クリア条件を所有します。RC3では10番目のキャンペーン位置と敵種別HP倍率を追加しました。

```ts
interface StageDefinition {
  id: string;
  titleKey: string;
  campaign?: { order: number; role: "standard" | "final" };
  arena: ArenaDefinition;
  obstacles: ObstacleDefinition[];
  encounterDeckId: string;
  enemyPoolId: string;
  difficulty?: StageDifficultyDefinition;
  clearCondition: ClearConditionDefinition;
  bossId?: string;
}
```

- コンテンツ定義はPhaser型を参照しない。
- `ArenaScene`や描画クラスへstage IDごとの分岐を増やさない。
- 不正なID、空の敵プール、到達不能な出現領域は起動前またはテストで失敗させる。
- モード、ステージ、シード、ルール版をラン記録へ残す。
- Endlessは同じ登録基盤を使うが、v0.7でルールを変更しない。

## 戦闘展開と安全規則

Encounter Directorは予告、実行、回復、完了、中断を明示し、弧、挟撃、護衛編成を決定論的に生成します。

- プレイヤーと出現地点の最低距離を守る。
- 障害物内、崩壊領域外、画面へ入れない位置を除外する。
- 侵入方向を予告し、実体出現までに移動判断の時間を置く。
- 安全候補がない場合は別方向へフォールバックし、成立しなければ延期する。
- 同一シードとworld状態から同じ地点列を得る純粋試験を持つ。

「展開地点を確保する」のように意味が不明確な文言は使いません。四方侵入、敵の役割、次に必要な行動を短い目的文で示します。

## CommanderとCharger

Commanderは最大HP 500の優先標的です。存在中は護衛増援と速度補正を発生させ、撃破すると圧力が下がります。Pulseは集束維持、Spreadは周囲の増援処理で異なる強みを使えます。

Chargerは`approach -> telegraph -> prepare -> charge -> recovery`を明示します。予告開始時に進路を固定し、回避後に反撃機会を作ります。画面外、障害物直前、同時実行数の安全制約を共有します。

## 最終ボス

`final-command-ship`はHP 3400で、プレイヤーを障害物経路に沿って追跡します。

- `targeted-salvo`: 現在位置を基準に広い扇状射撃を予告し、射線離脱を要求する。
- `escort-pincer`: 護衛を挟撃配置し、抑制斉射を重ねる。
- HP 50%で第2段階へ入り、移動、弾数、速度、護衛数を増やす。
- ボス中も通常ウェーブを継続し、最終ウェーブの出現頻度を維持する。
- 開始遷移フレームの通常敵と敵弾だけを整理し、その後は盤面を空にしない。
- ボス本体、護衛、通常敵、敵弾は既存上限を共有する。

固定fixtureでは攻撃の予告、障害物による射線遮断、移動、段階変化、撃破の一回性を確認します。画面外からの不可視攻撃、無敵待ち、予告なしの高密度攻撃は採用しません。

## UI/UX

タイトルからEndlessと最終遠征を分けて開始できます。プレイ中は次を表示します。

- 第10ステージ、現在Act、短い目的。
- Encounterの種類、侵入方向、開始までの予告。
- Commanderの現在HPと最大HP。
- ボスHP、段階、攻撃予告。
- 勝利または敗北、到達Act、クリア時間、最後のダメージ源。

目的文、HP、XP、スコア、警告を重ねません。色だけで危険方向や段階変化を伝えず、文字、形、音を併用します。

## 音楽と世界観

勝利時だけ遠征専用BGMへ切り替え、敗北SEは重ねません。ステージ別通常曲とボス曲は、複数ステージ実装時に`StageDefinition`から解決できるよう拡張します。

世界観は未確定です。四方から侵入する敵、障害物、追跡する指揮中枢を自然に説明できることを条件とし、「敵艦隊」「魔物」などの固有テーマは戦闘設計と分けて採否します。

## 記録と採否

最低限記録するもの:

- `modeId`、`stageId`、campaign order、Act、勝敗、クリア時間。
- カードの抽選、開始、完了、中断と理由。
- Commanderの出現、撃破、存在時間。
- ボス攻撃種、段階、攻撃別被弾、撃破または残HP。
- 最大敵数、弾数、Pickup数、意味のある局面の最長空白。
- 同一シードのイベントhashと終了world hash。

RC5の早送り確認では、3 fixed seed x 2武器の6構成がすべて勝利しました。RC6では全6構成のCommander撃破、Act 5、全攻撃種と、専用fixtureのboss phase 2を機構到達性としてassertします。自然runは両武器のphase 2到達、3/6以上かつ各武器1勝以上を戦闘退行の監視値にします。CommanderのAct境界越えは専用fixture、spawn deferと終了worldは全入力replayで保証します。自動入力は到達不能、決定論、極端な密度の回帰検出に使い、面白さと人間の難度判定は通常UIの手動ランで行います。

Expeditionの主記録は勝利の総クリア時間です。速攻成果は戦術点へ重ねず時間メダルにし、敗北は履歴へ残しても勝利PBを上書きしません。詳細は[RC6の時計と記録規則](../../engineering/expedition-rc6-clock-and-ranking-adr/)を正本とします。

## 対象外

- Stage 1と5、deferredのStage 2から4、6から9の実装。
- ステージ選択、進行保存、ボス練習。
- 外部アカウント、クラウド同期、オンラインランキング。
- 3つ目の開始武器、装備ドロップ、恒久能力上昇。
- 長編ストーリー、会話、ボイス、最終アート。
- モバイル向けタッチ操作の正式対応。

## 採用ゲート

1. 1ランを「四方警戒、重装襲来、Commander、高速・射撃包囲、最終決戦」と説明できる。
2. 敵の初出順が`赤 -> 大型黄 -> 小型黄緑 -> 紫`で固定される。
3. ボス戦でも通常圧力が続き、道中より明確に難しい。
4. RC6の自動6構成が全ボス攻撃へ到達し、phase 2 fixture、3/6以上かつ両武器で自然勝利があり、通常UIでもPulseとSpread双方が到達不能でない。
5. 予告を見て射線を外すか障害物を使う回避機会がある。
6. 既存Endlessのルール、履歴、ランキング、入力を回帰させない。
7. 同一シードの再現、ランJSON、fixture、手動ランから失敗理由を調査できる。

Stage 1 / 5 / 10の実装順と残りstageの延期条件は[エクスペディション3作戦検証](../expedition-campaign/)を正本とします。RC6の検証手順は[RC6 QA・採否計画](../../playtest/v07-rc6-qa-plan/)を参照してください。
