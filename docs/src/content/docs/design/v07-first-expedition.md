このページはv0.7の体験要件の正本です。時間、HP、出現数などの数値は初期仮説であり、固定シード試験と手動プレイで調整します。世界観の固有名詞と最終アートはまだ固定しません。

## 製品仮説

v0.7では、現在のエンドレスへコンテンツを横に増やすのではなく、開始から勝利までの起伏を持つ最初のエクスペディションを1本完成させます。

> 読める襲来へ位置取りと標的優先で対応し、ビルドを完成させ、最後に敵指揮艦を撃破する。1ランの中で準備、危機、反撃、決戦を体験できる8分から10分の勝利型モード。

この縦切りで検証するのは、ステージ数や敵数ではなく、構造化した展開によって同じ操作基盤にドラマと再挑戦理由が生まれるかです。

## モードの位置づけ

| モード | 目的 | 終了 | v0.7での扱い |
| --- | --- | --- | --- |
| Last Stand / Endless | 防衛線が崩壊するまで戦果と生存時間を伸ばす | 撃墜または安全領域消失 | 現行ルールを回帰基準として維持 |
| Counterattack / Expedition | 襲来を突破して敵指揮艦を撃破する | ボス撃破で勝利、HP 0で敗北 | 最初の縦切りを実装 |
| Challenge | 固定条件で技能評価を競う | 条件達成または撃墜 | v0.8以降 |

Last StandやCounterattackなどの仮称は実装上の安定IDに使いません。内部IDは`expedition`、最初のステージは既存の`arena-default`をデータ定義へ移したものから始めます。表示名は世界観決定後に変更可能とします。

## プレイヤーへの約束

1. 30秒から90秒ごとに、位置取り、標的優先、回収、強化のいずれかを考え直す局面が来る。
2. 襲来方向と危険攻撃は、被弾する前に形、位置、音の2系統以上で読める。
3. Pulseは単線の精密射撃と優先標的処理、Spreadは扇状の群れ処理という役割を最後まで維持する。
4. 失敗時は、どの局面、敵、攻撃で崩れたかをリザルトから説明できる。
5. 同じシードは同じ戦況を再現するが、複数の戦闘展開カードにより別シードを試す理由がある。

## 1ランの構成

時間帯は調整用の初期範囲です。イベントを秒数だけで直接分岐せず、ActとEncounter Directorが進行を所有します。

| Act | 初期時間帯 | 役割 | 最低限の内容 |
| --- | --- | --- | --- |
| 1 配備 | 0:00から1:30 | 操作と初期ビルドの準備 | 基本敵、最初の強化、低圧の構造化出現 |
| 2 第一襲来 | 1:30から3:00 | 初めて展開を読む | 方向予告付きカード1枚、イベント後の短い回収時間 |
| 3 反撃準備 | 3:00から4:30 | ラン方針を固める | 指揮艦エリートまたは優先標的、既存契約の再利用判断 |
| 4 防衛線突破 | 4:30から6:30 | 複合圧力へ対処する | 異なるカード2枚、予兆付き突進敵、短い回復機会 |
| 5 指揮艦決戦 | 6:30から9:00 | ビルドと技量を統合する | ボス、2種以上の攻撃、段階変化、専用HP表示 |
| 猶予 | 9:00から10:00 | 長期化を終わらせる | 読める強化状態。即死や不可避攻撃にはしない |

通常ビルドが4分から6分で完成する現在の曲線を利用し、ボス戦で通常最終強化または序盤EXを使える構成を狙います。エクスペディション専用の恒久強化は追加しません。

## 最小コンテンツ

v0.7の完了に必要な量を次へ固定します。

- 勝利条件を持つエクスペディション1モード。
- 既存アリーナをデータ定義へ移したステージ1件。
- 構造化した出現形状3種以上。候補は弧、挟撃、護衛編成。
- 戦闘展開カード3枚以上。既存の射撃体集中、高速群襲来、重装包囲を再利用してよい。
- 優先して倒す意味を持つ指揮艦エリート1種と、エリート特性1件。
- 移動判断を試す予兆付き突進敵1種。
- 2種以上の基本攻撃と1回以上の段階変化を持つボス1体。
- Act、目的、ボスHP、勝利または敗北を伝える最小UI。
- 背景1種、敵またはボス1種、危険イベント1種の視覚縦切り。

第2ステージ、3つ目の武器、複数ボス、装備ドロップは完了条件へ含めません。

## ステージ要件

`StageDefinition`は少なくとも次を持ちます。

```ts
interface StageDefinition {
  id: string;
  titleKey: string;
  arena: ArenaDefinition;
  obstacles: ObstacleDefinition[];
  encounterDeckId: string;
  enemyPoolId: string;
  clearCondition: ClearConditionDefinition;
  bossId?: string;
}
```

- コンテンツ定義はPhaser型を参照しない。
- `ArenaScene`や描画クラスへstage IDごとの分岐を増やさない。
- 不正なID、空の敵プール、到達不能な出現領域は起動前またはテストで失敗させる。
- モード、ステージ、シード、ルール版をラン記録へ残す。
- Endlessは同じ登録基盤を使うが、v0.7で挙動を変更しない。

## 戦闘展開カード

カードは「いつ」「何を」「どこから」「何のために出すか」をデータとして持ちます。

最低限の項目:

- 安定ID、表示キー、タグ、利用可能Act。
- 予告、実行、回復の各時間。
- 出現予算、敵構成、出現形状。
- 必要脅威度、クールダウン、直前カードとの排他条件。
- 成功、失敗、中断条件と任意の報酬。
- 使用する乱数ストリーム。

Directorはシャッフルバッグまたは重み付き抽選を使い、同じカードの連続と同じ侵入方向の反復を制限します。抽選結果を先に確定し、描画やサウンドが乱数を消費しない状態を維持します。

## 出現と安全規則

構造化出現は見た目の整列より、回避可能性を優先します。

- プレイヤーと出現地点の最低距離を定義する。
- プレイヤーを囲む全方向へ同時に不可避接触を作らない。
- 障害物内、崩壊領域外、画面へ入れない位置を除外する。
- 進入方向を予告し、実体出現までに最低1回の移動判断ができる時間を置く。
- 安全候補がない場合は別形状へフォールバックし、それも失敗したら出現を延期する。
- 同じシードとworld状態から同じ地点列を得る純粋試験を持つ。

## エリートと突進敵

指揮艦エリートは単なる高HP敵にしません。存在中に増援頻度、周囲の防御、射撃支援のいずれか1つを強め、倒すと圧力が明確に下がる役割を持たせます。

予兆付き突進敵は次の状態を明示します。

1. 照準または進路予告。
2. 短い準備。
3. 直線突進。
4. 外した後の回復時間。

障害物との関係、画面外からの開始、複数同時発生に上限を置きます。予告を見て避けた結果、反撃機会が生まれることを完了条件とします。

## ボス要件

最初のボスは武器別DPS検査ではなく、照準維持、群れ処理、位置取りを順に試します。

- 基本攻撃を2種以上持つ。
- 各攻撃は`telegraph -> execute -> recovery`を明示する。
- HPまたは時間で1回以上段階が変わる。
- 雑魚を出す場合も画面密度上限と出現安全規則を使う。
- Pulseが優先部位または射線を維持する時間を作る。
- Spreadが随伴敵または複数部位を処理する時間を作る。
- どちらの武器でも、固定fixture上で全攻撃を無被弾回避できる。
- 敗北時に最後のボス攻撃種を記録する。

画面外からの不可視攻撃、長時間の無敵待ち、常時接触ダメージで近づけない状態は採用しません。

## UI/UX要件

### 開始前

- タイトルからEndlessとExpeditionを区別して選べる。
- 未実装モードを並べない。
- 選んだモード、ステージ、武器を開始前に確認できる。
- 既存の2操作以内でEndlessを始められる導線を壊さない。

### プレイ中

- 現在Actと短い目的を、既存HP、XP、スコアより低い頻度で表示する。
- 戦闘展開の予告は方向、時間、種類を示し、HUDや強化選択と重ねない。
- ボス戦だけ専用HPと段階を上部へ表示する。
- 色だけで危険方向、準備完了、段階変化を伝えない。
- 警告文はプレイ領域を長時間覆わず、可変長でも重ならない。

### リザルト

- 勝利と敗北を最初に区別する。
- 到達Act、クリア時間、ボス残HPまたは撃破、最後のダメージ源を表示する。
- モード、ステージ、武器、シード、ルール版を確認できる。
- ExpeditionとEndlessを同じランキング条件へ混ぜない。
- v0.7ではクリア時間を表示するが、オンライン競争や速度ランキングは導入しない。

## 世界観と視覚の境界

艦隊防衛シミュレーションを第一候補にします。v0.7では次だけを固定します。

- プレイヤーは防衛側の機動兵器または迎撃艇である。
- エクスペディションは敵艦隊の指揮系統を破壊する反撃作戦である。
- Actの変化は戦況表示、背景、境界、音の節目として揃える。
- 指揮艦は通常敵と輪郭、サイズ、コア、警告音で区別する。

有人機か無人機か、戦闘が現実かシミュレーションか、固有名詞、人物会話は確定しません。長い会話、カットシーン、プレイを止める説明も追加しません。

## 記録と成功指標

最低限追加する計測:

- `modeId`、`stageId`、Act、勝敗、クリア時間。
- カードの抽選、開始、完了、失敗、中断と理由。
- 出現形状、侵入方向、安全候補の棄却回数。
- エリート出現、撃破、存在時間、支援効果。
- ボス攻撃種、段階、攻撃別被弾、撃破または残HP。
- 意味のある局面間隔と最長空白。
- 既存の武器、被ダメージ、フレーム性能指標。

初期KPI:

| 指標 | v0.7の判定 |
| --- | --- |
| 展開間隔 | 意味のある局面の最長空白90秒以内を目標。120秒超は不採用 |
| 再現性 | 同一シード、同一入力列の終了hashが完全一致 |
| 出現安全 | 固定fixtureで不可避出現0件 |
| ボス可避性 | 両武器で全攻撃の回避fixtureが通る |
| 手動到達性 | PulseとSpreadで各3ラン以上。両方でボス到達、少なくとも一方でクリア |
| 武器役割 | Pulseの優先標的成果とSpreadの複数標的成果を別指標で確認 |
| 説明可能性 | 敗北理由をリザルトとランJSONから攻撃種まで追える |
| 性能 | 現行のフレーム、敵、弾、pickup上限を超えない |

クリア率や武器別クリア時間の厳密な均衡は、少数ランだけで固定しません。自動入力は回帰と到達不能の検出に使い、面白さや人間の技能上限の判定器には使いません。

## 対象外

- 外部アカウント、ログイン、クラウド同期。
- オンラインランキング、サーバー側スコア検証。
- 恒久的な能力値上昇や課金。
- 第2ステージ、ランダム地形、動的NavMesh。
- 3つ目の開始武器、装備ドロップ、アイテム量産。
- 長編ストーリー、会話システム、ボイス。
- モバイル向けタッチ操作の正式対応。
- 観戦AIの再調整または既定化。

対象外の要求はv0.7のIssueへ混ぜず、v0.8以降の候補へ送ります。

## 採用ゲート

v0.7を完了とするには、機能が接続しただけでなく次を満たします。

1. 1ランを「配備、第一襲来、反撃準備、突破、指揮艦決戦」と説明できる。
2. 既存Endlessのルール、履歴、ランキング、入力を回帰させない。
3. 予告を見て行動を変える場面と、優先標的を倒して圧力を下げる場面がある。
4. PulseとSpreadが異なる得意行動を持ったままボスへ到達できる。
5. 同じシードの再現、ランJSON、固定fixture、手動ランから失敗理由を調査できる。
6. 新しいランをもう一度試したい理由を、カード順、武器、プレイ改善のいずれかで説明できる。

このゲートを満たさない場合は、第2ステージやメタ進行へ進まず、カード、Act、ボスのいずれが原因かを計測して戻します。
