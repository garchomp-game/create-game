## 目的

障害物を単なる移動妨害ではなく、敵の誘導、射線管理、跳弾を通じて判断を増やす地形として機能させます。

対象チケット:

- `PH-V04-005`: 障害物配置と弾の壁接触見直し。
- `PH-V06-004`: Pulse最終強化「反響回路」。
- `PH-V06-014`: 障害物対応の敵ナビゲーション。
- `PH-V06-019`: Pulse高速弾、連続衝突判定、外周反射。

## 現在の配置

- 開始位置付近の移動と射線を塞いでいた中央ブロックは削除済みです。
- 残る障害物は固定配置で、ラン中には移動・生成・破壊しません。
- プレイヤー、敵、弾、敵弾、ピックアップは共通の円対矩形衝突規則を使います。

動的地形と自動生成配置は、ステージ設計と検証済み配置の仕組みができるまで対象外です。

## 敵ナビゲーション

v0.6.2では、敵ごとに毎フレームA*を実行せず、プレイヤー位置を終点とする共有Dijkstra経路場を使います。

処理順:

1. 敵からプレイヤーまで、敵半径を含む直線が障害物と交差しないか調べる。
2. 直線が通る場合は、従来どおりプレイヤーへ直接進む。
3. 塞がれている場合は、24pxグリッドの共有経路場から迂回方向を得る。
4. 経路のうち現在位置から直接到達できる最も先の地点を選び、格子状の蛇行を減らす。
5. 通行可能セルまたは経路が得られない場合は、従来の直接移動と軸別衝突へ戻す。

経路セルの通行判定には敵半径に2pxの余白を加えます。実際の直線移動は敵半径で判定し、すでに壁際へ到達した敵が余白の内側から出られなくなる状態を避けます。

経路場はプレイヤーの目標セルと敵半径ごとに共有します。同じ条件の敵は同じ場を使い、最近使った目標セルを最大64件保持します。これにより、障害物を回る判断を決定論的に保ちながら、敵ごとの探索を避けます。

射撃体は弾半径で射線を確認します。射線が切れている間は経路場でプレイヤーへ近づき、障害物越しには発射しません。近すぎる場合の後退と、適正距離での停止は従来どおりです。

現時点では敵同士の局所回避、隊形、動的障害物、複数目標は扱いません。これらは構造化出現とステージを導入した後に、必要性をプレイで確認して追加します。

## 弾と射線

プレイヤー弾:

- 1フレームの移動を線分として扱い、敵円、障害物、アリーナ外周との最初の交差位置を求める。
- 貫通時は敵配列の順ではなく、線分上の交差時刻が早い敵から`hitCapacity`まで命中する。
- `ricochetRemaining > 0`の場合は障害物で反射し、残数を1減らす。
- 反射後は残り寿命を最大0.35秒に制限し、同じフレームの残り時間も復路へ使う。
- `ricochetRemaining === 0`で障害物へ当たると消える。
- Pulseの反響回路を取得すると、障害物または固定アリーナ外周で使える1回分の跳弾が付く。
- 外周反射はPulse専用で、Spread、敵、敵弾、プレイヤー、敵出現位置、崩壊の安全領域には作用しない。

Pulseの加速弾は1ランク22%で、最大約1405px/sです。50msの最大ステップでは約70px進むため、終点の円衝突だけでは半径11pxのFastを飛び越えます。連続衝突判定は弾速強化を成立させるためのルールであり、描画補間だけの処理ではありません。

敵弾:

- 障害物へ当たると消える。
- 射撃体は発射前にも射線を確認するため、遮蔽物越しには新しい敵弾を作らない。

## 計測

詳細ランJSONの`stats.navigationMetrics`へ次を記録します。

| 項目 | 意味 |
| --- | --- |
| `directFrames` | 直線追跡または射撃位置維持を使った敵フレーム数 |
| `pathFrames` | 経路場による迂回を使った敵フレーム数 |
| `fallbackFrames` | 経路を得られず従来移動へ戻した敵フレーム数 |
| `fieldBuilds` | 新しい目標セル・敵半径の経路場を作った回数 |

`summary.tsv`と`summary.csv`には4値と迂回比率を出します。プレイヤー向け`RunRecord`には経路診断を保存せず、ランキングの比較条件だけを`rulesetVersion`で分離します。

反響回路は、障害物反射、外周反射、左・右・上・下の外周面別反射、跳弾後命中を`RunRecord`と詳細ランJSONへ保存します。`summary.tsv`と`summary.csv`にも同じ列を出し、旧記録は追加項目を0として読み込みます。

## 自動確認

- 射線が通る敵は直接追跡する。
- 同じ状態では同じ迂回方向を選ぶ。
- 追跡体が障害物内へ入らず反対側へ回れる。
- 同じ半径の敵が経路場を共有する。
- 最近訪れたプレイヤーセルへ戻ると経路場を再利用する。
- 射撃体が遮蔽物越しに撃たず、射線を取り直す。
- 機能フラグを無効にすると従来の軸別衝突へ戻る。
- 最大Pulse弾速と50msステップでFastを横切っても命中する。
- 貫通弾が手前から順に命中し、同一フレームの往路と復路で異なる敵へ命中できる。
- Pulse外周反射の無効化と、Spreadが外周反射しないことを固定する。

## 手動確認

- 敵が角で不自然に震えたり、全員が同じ一点へ詰まったりしないか。
- 障害物を挟むことが完全な安全地帯ではなく、一時的な距離と射線の利点になるか。
- 以前より包囲圧力が上がっても、プレイヤーが移動経路を読み直せるか。
- 射撃体が射線を取り直す動きに、理不尽な近道や長い停止がないか。
