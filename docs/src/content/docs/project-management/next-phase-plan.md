## 現在の判断

v0.6「ビルドの個性とエンドレス後半」は、実装、固定10シードの武器比較、通常E2E、画面比較、実時間15分耐久まで完了しました。

現在の残作業は手動採否判定です。`pulse`と`spread`を各2本以上プレイし、開始武器、反響回路、射撃体集中、後半契約が実際の狙い方と位置取りを変えるか確認します。手動所感を得るまではv0.7実装へ進みません。

- v0.6の個別要件は[v0.6チケット詳細](../v06-tickets/)を正本とする。
- v0.6以降のつながりは[中長期作業計画](../gameplay-expansion-plan/)を参照する。
- ゲームプレイ原則は[拡張設計の全体像](../../design/gameplay-expansion-blueprint/)を参照する。

## v0.6で解く問題

### 1. ビルドが同じ形へ収束して見える

v0.5の手動5ランはすべて開始武器が`pulse`でした。全ランで`splitShot`が最大ランク2、`vitalCore`はランク0で、直近の約301秒と322秒の2ランは最終強化構成が同一です。

これは直ちにバランス不良を意味しません。現在は全提示履歴を保存していないため、「候補に出なかった」と「出たが選ばなかった」を区別できません。先に計測を追加し、その後に候補重みや効果を調整します。

### 2. 後半ほど意味のある選択が減る

v0.5ではXP必要量が`growth: 1.4`で増え、全強化後も空のレベルアップが発生していました。

v0.6では`growth: 1.25`、上限140へ変更し、全強化後はXPを停止してビルド完成を明示しました。

### 3. 武器差をスコア以外で説明できない

武器別計測へ命中ボレー率、1射で命中した異なる敵数、敵種別命中・撃破、移動、反響回路成果を追加しました。自動入力では役割差が出ていますが、人間の操作差は未確認です。

### 4. 90秒以降の新しい判断が少ない

135〜165秒の射撃体集中と240秒の後半契約を追加しました。予告理解と危険・報酬判断が手動プレイで成立するかを確認します。

### 5. 戦闘ルールに未確定箇所がある

- `hitCapacity`は1発の総命中可能数として表示と実装を統一した。
- 偶数の分裂弾は照準中央を空けて左右へ飛ぶ。
- 無敵時間中に重なった敵弾は、ダメージを与えず消滅する。

偶数分裂と敵弾消費は既存挙動を維持し、固定配置試験でルールを固定しました。

## 実施結果

### 段階A: ルールと比較契約

1. `PH-V06-010` 弾・貫通・無敵時間ルールを監査する。
2. `PH-V06-008` 強化履歴と武器差の計測契約を定義する。
3. [`PH-V06-009` #12](https://github.com/garchomp-game/create-game/issues/12) v0.6で変更する画面とデバッグ責務だけを分割する。

`PH-V06-009`は全面リファクタリングではありません。開始武器選択、ビルド要約、開発計測の追加で実際に触る責務だけを抽出します。

### 段階B: 再現性と成長間隔

4. `PH-V06-007` 出現、強化、ドロップ、戦闘展開の乱数列を分離する。
5. `PH-V06-001` XP曲線、強化提示間隔、全強化後の状態を再設計する。

XPは現行式と候補式を固定シードで比較し、感覚だけで変更しません。

### 段階C: ビルド縦切り

6. `PH-V06-002` PulseとSpreadの開始武器選択を接続する。
7. `PH-V06-003` 強化分類と効果合成を必要範囲だけ分離する。
8. `PH-V06-004` Pulseの跳弾を第一候補に、挙動変化型の最終強化を1つ試す。
9. `PH-V06-005` 強化比較とビルド履歴を一時停止・リザルトへ接続する。

`pierce`は固定配置と開発計測で役割を確認し、通常開始武器へ出すかをこの段階の最後に判断します。

### 段階D: エンドレス後半

10. `PH-V06-006` 120秒から180秒へ予兆付き危険イベントを1つ追加する。
11. 危険イベントが成立した場合だけ、約240秒の危険・報酬契約を試す。

現在の手動ランは最長でも約322秒なので、最初の判断を6分以降へ置かず、通常ランでも検証可能な時刻にします。

### 段階E: 採否判定

12. `PH-V06-008` PulseとSpreadを固定条件で各10試行以上比較する。
13. `PH-V06-011` 武器ごとに最低2本の手動ランを行い、v0.6の採否を判定する。

## 比較指標

スコアだけで武器や強化を評価しません。

- 強化の提示順、選択順、時刻、選択間隔。
- 1射あたりの異なる命中敵数と敵種。
- 毎分スコア、毎分撃破、被ダメージ源。
- 移動距離と危険イベント前後の移動方向。
- 最終強化の取得時刻、発動回数、追加命中。
- 開始武器別の生存時間分布。
- 手動所感として、照準距離、位置取り、標的優先が変わったか。

20%以上の一貫した差は強い採用信号としますが、数値差だけで面白さを断定しません。

## 残る手動ゲート

| 本数 | 武器 | 確認すること |
| ---: | --- | --- |
| 2本以上 | Pulse | 反響回路取得前後で障害物を狙うか、継続照準と標的切り替え |
| 2本以上 | Spread | 近距離・広角を選ぶ位置取り、複数敵への角度、被弾の納得感 |
| 全体 | 両方 | 短時間、通常、長めを含め、射撃体集中と後半契約へ最低1回到達 |

各ランは通常UIから実施し、ブラウザ履歴と`phaser/logs/runs`へ保存します。固定シード比較を追加する場合は`?seed=20260619`を使います。

## v0.6の終了条件

- PulseとSpreadの違いを、操作と強化方針の一文でそれぞれ説明できる。
- 長時間ランでも、成長が理由なく止まったように見えない。
- 挙動変化型の最終強化が、狙い方または地形利用を変える。
- 同じシードと開始武器で主要イベント列を再現できる。
- リザルトと履歴で、開始武器、強化順、最終強化を比較できる。
- 予兆付き危険イベントに対する反応を計測・説明できる。
- 実験ルール、デバッグ、自動テストが通常ランキングへ混ざらない。
- 自動比較、手動ラン、E2E、画面比較から採用・調整・削除を判断できる。

ステージ、エリート、ボスはv0.7で、戦闘展開制御と出現安全規則を作ってから扱います。
