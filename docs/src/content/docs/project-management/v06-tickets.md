## フェーズ仮説

v0.6で検証する中心仮説は次の1つです。

> 開始武器、成長選択、挙動変化型の最終強化を接続すると、同じアリーナでも狙い方、位置取り、標的優先が異なるランになる。

v0.6.1では、長時間プレイの所感から次の補助仮説を追加します。

> 通常ビルド後の緩やかな成長と、それを上回る脅威・戦場収縮を並行すると、後半にも選択を残しながら永久生存を防げる。

v0.6.2では、後半の操作と選択の単調さに対して次を追加します。

> 敵が障害物を迂回し、EX候補がサイクル内で重複しなければ、地形利用と取得順の判断を増やしつつ、同じ強化の連打を防げる。

v0.6.3では、成長後に武器形状が収束する問題へ次を追加します。

> Pulseを単線集中、Spreadを広角複数標的として固有成長と最終強化を対称に用意すれば、総合性能を近づけたまま照準と位置取りを分けられる。

v0.6.4では、手動比較でPulseだけ通常ビルド完成へ届かなかった結果から次を追加します。

> Pulseへ分裂を戻さず、高弾速、貫通順、障害物・外周反射を接続すれば、単線形状のまま操作精度を処理量へ変換できる。

v0.6.5では、手動26355点ランで反射後命中の寄与元が分からなかったため次を追加します。

> ルール値を固定したまま反射成果の出所と実フレーム性能をラン単位で保存すれば、公開ベータの調整を推測ではなく実測から始められる。

大量の武器、敵、スキルを先に追加するのではなく、開始前の選択からリザルト比較までを1本通して検証します。

## 現在の根拠

v0.5の通常UIから記録した手動5ランは、約98秒から322秒、3211点から18742点でした。

- 全5ランが開始武器`pulse`で、`spread`と`pierce`の通常プレイ実績はない。
- 全5ランで`splitShot`がランク2へ到達し、`vitalCore`はランク0だった。
- 直近の約301秒と322秒の2ランは、6強化の最終ランク構成が同一だった。
- これだけでは`vitalCore`が提示されなかったのか、意図的に選ばれなかったのかを区別できない。
- 現在のログには、強化の全提示履歴、選択順、選択間隔、1射で命中した異なる敵数が残らない。
- 全強化が最大になると、レベルは上がるが選択画面は出ず、報酬のないレベルアップになる。
- 無敵時間中にプレイヤーへ重なった敵弾は、ダメージを与えず消滅する。

したがって、武器やXPを調整する前に、比較指標と戦闘ルールを固定します。

## スコープ判断

### v0.6で成立させるもの

- 製品画面では、まず`pulse`と`spread`を開始武器として比較する。
- `pierce`は削除せず、固定配置テストと開発計測で役割を確認してから通常選択へ出すか判断する。
- v0.6では挙動を変える最終強化をPulseへ1つ試作し、v0.6.3で同じ設計密度のSpread最終強化へ拡張する。
- 既存敵を使った予兆付き危険イベントを1つ試し、成立後は3種の反復デッキへ拡張する。
- 通常ビルド完成後のEX進行、脅威ティア、アリーナ崩壊をv0.6.1の是正範囲として扱う。
- 障害物対応の共有敵経路と循環型EXデッキをv0.6.2の是正範囲として扱う。
- 武器固有成長、役割別の後半HP、高解像度DOM選択UIをv0.6.3の是正範囲として扱う。
- Pulse高速弾、連続衝突判定、外周反射をv0.6.4の手動比較是正として扱う。
- 障害物 / 外周別の追撃成果と実フレーム性能をv0.6.5の計装として扱う。
- 開始武器、強化履歴、最終強化、主要指標をリザルトと開発ログで比較する。

### v0.6で成立させないもの

- 3武器すべての最終強化完成。
- 大量の新規強化、新規敵、アイテム追加。
- ステージ、エリート、ボスの本実装。
- 防御ダッシュ、右クリック固有スキル、オンラインランキング。
- 武器間の完全な競技バランス。

## 実装・自動検証結果

2026-07-14時点で、v0.6.4の実装・手動採否とv0.6.5の計装を完了しました。フェーズ全体を完了とし、600秒以降の追加所感は公開ベータの非停止観測へ移します。

実装した主な規則:

- アプリ版を`0.6.5`へ更新し、数値不変のためルール版は`phaser-v0.6.4-pulse-ballistics`を維持した。
- 出現、強化、ドロップ、戦闘展開、ステージ候補を`arena-rng-v1`の独立乱数列へ分けた。
- XP式を`growth: 1.25`、`maxXp: 120`へ調整し、PulseとSpreadはともに25回の選択、4〜6分で`buildCompletedAt`へ到達する。
- 開始武器はPulseとSpreadだけを表示し、Pierceは製品選択から外したまま固定配置試験へ残した。
- 強化を武器、機動、生存、補助、最終強化へ分類し、効果合成の由来を開発ログへ出力する。
- Pulseの武器系強化7ランクで最終強化「反響回路」を解放し、障害物で1回跳弾する。反射後寿命は0.35秒とした。
- Pulseは0.12秒の単線連射と「集束共鳴」、Spreadは3発から5発の「分裂射撃」と最終強化「掃射循環」へ分けた。
- Pulseの加速弾だけを1ランク22%へ上げ、最大約1405px/sとした。Spreadと開発用Pierceは15%を維持した。
- プレイヤー弾を移動線分で連続判定し、貫通を交差時刻順、反射前後を同一フレームで処理する。
- 反響回路は障害物または固定外周で合計1回反射する。外周はPulse弾以外と崩壊領域に作用しない。
- 15分耐久で同時弾408を検出したため、射撃間隔を変えず同時敵弾を256で制限し、合計弾数300以下の可読性上限を維持した。
- 240秒以降の敵HP成長をFast、Chaser、Ranged、Bruteの役割別倍率へ分けた。
- 開始武器、通常 / EX強化、契約をDOMボタンへ移し、縦画面では1列表示する。
- 射撃体集中はシードから135〜165秒へ配置し、6秒予告、18秒集中、8秒回復で構成する。
- 240秒の後半契約は、標準維持と、敵速度1.12倍・スコア1.3倍の過負荷を選ぶ。過負荷ランは`nonStandardRuleset`として通常ランキングから除外する。
- 通常ビルド完成後はEXレベルへ移り、4候補を同一サイクル内で重複させず、最後の1件を自動取得する。
- 過給サイクルは+10%を5段階、超過駆動は+6%を5段階とし、攻撃力と最大HPは上限なく残す。
- 敵は射線が通れば直接追跡し、障害物で塞がれる場合は共有Dijkstra経路場で迂回する。射撃体は遮蔽物越しに発射しない。
- 脅威ティアは240秒から45秒ごとに上昇し、敵HPと攻撃力は上限なく、画面密度に関わる値は安全上限まで増える。
- 射撃体集中、高速群襲来、重装包囲を決定論的なシャッフルバッグから反復する。
- 600秒からアリーナを45秒ごとに縮小し、領域外ダメージと最終的な安全領域消失で有限終了を作る。
- 1発の総命中可能数を`hitCapacity`、残り命中数を`hitsRemaining`へ統一した。無敵時間中に重なった敵弾は消費する既存ルールを維持した。
- 反射後命中・撃破を障害物と外周へ帰属し、外周追撃命中を面別にも保存する。
- 実フレーム時間の標本数、平均、p95、最大、50ms超過数、推定FPS、Phaser実測FPSをランJSONとCSV / TSVへ保存する。

## 共通の着手条件

- 変更前の固定シード結果を保存する。
- 変更するルールと`rulesetVersion`への影響を明記する。
- プレイヤー履歴へ残す値と、開発ログだけへ残す値を分ける。
- 通常ランキングへデバッグ、自動テスト、実験ルールを混ぜない。
- UI変更では960 x 540と狭いブラウザ表示を確認する。
- 数値変更には、比較する仮説と採用、調整、撤回の条件を付ける。

## PH-V06-001 XP曲線と強化選択間隔の再設計

- GitHub: [#13](https://github.com/garchomp-game/create-game/issues/13)
- 優先度: P0
- 依存: `PH-V06-007`、`PH-V06-008`の計測契約

目的:

長時間ランで成長が止まったように見える状態を解消し、意味のある選択を一定間隔で提示します。

対象:

- レベル別必要XP、実取得XP、強化提示間隔の開発計測。
- 現行`growth: 1.4`と、区間式または上限付き候補の比較。
- 全強化到達後の`buildComplete`状態と、その後のXP表示規則。
- 最大ランク候補を除外した後も、空の選択を報酬として扱わない処理。

受け入れ条件:

- 最初の強化が5秒から10秒を目安に発生する。
- 開始2分までは15秒から30秒、以降は30秒から60秒を目安に意味のある選択が発生する。
- 10分ランで`longestMeaningfulChoiceGap`が90秒を大きく超えない。
- 全強化後は、報酬のないレベルアップ演出や空の選択を出さない。
- 現行式と候補式を固定シードシミュレーションで比較し、変更理由を数値で残す。

対象外:

- 恒久成長、装備レベル、アカウント進行。

全敵を一律に硬くせず、240秒以降に雑魚と高耐久標的を分け、Pulseの集中処理とSpreadの群れ処理を共存させます。

受け入れ条件:

- 45秒ごとのHP倍率をFast 1.10、Chaser 1.12、Ranged 1.15、Brute 1.17とする。
- 基礎HP、240秒以前、敵ダメージ、XP値を変更しない。
- 600秒付近でRangedは約7、Bruteは約11HPになり、Chaser / Fastは雑魚役割を維持する。
- Pulse / Spreadの生存、毎分撃破、毎分スコア差を各20%以内に保つ。
- Spreadの命中斉射あたり異なる敵数をPulseより15%以上高く保つ。
- 通常ビルド完成、敵上限滞在、経路負荷、接触死を自動・手動で監視する。

## PH-V06-019 Pulse高速弾と外周反射

- GitHub: [#30](https://github.com/garchomp-game/create-game/issues/30)
- 優先度: P0
- 依存: `PH-V06-008`、`011`、`013`

目的:

Pulseへ分裂を戻さず、高弾速、最大4体貫通、障害物・外周反射を使いこなす高技能武器として成立させます。

受け入れ条件:

- 加速弾はPulseだけ1ランク22%とし、最大約1405px/sにする。SpreadとPierceは15%、Pulseの連射は0.12秒を維持する。
- 弾移動線分と敵円を連続判定し、最大速度・50msステップでFastを横切っても命中する。
- 貫通は線分上の交差時刻順に処理し、同一フレームで往路命中、外周反射、復路命中が成立する。
- 反響回路取得後だけ、Pulse弾が障害物または固定外周で合計1回反射する。
- 外周はSpread、敵、敵弾、プレイヤー、画面外出現、崩壊領域へ作用しない。
- 同時敵弾を256以下、プレイヤー弾との合計を300以下に保ち、上限中も射撃タイマーへ発射待ちを蓄積しない。
- 障害物、外周、外周面別の反射数と跳弾後命中を履歴、JSON、CSV / TSVへ保存し、旧記録を0で移行する。
- 反射フィールドと外周反射地点を色だけに依存せず短い形状差で表示する。
- 固定10シードの主要武器差を20%以内に保ち、次のPulse手動3ラン中1本以上で240秒から360秒の通常ビルド完成を確認する。

対象外:

- Pulseの分裂、撃破時破片、飛行中の物理加速、敵や敵弾を止める物理外壁。

## PH-V06-020 反響回路の成果帰属と実測性能ログ

- GitHub: [#31](https://github.com/garchomp-game/create-game/issues/31)
- 優先度: P1
- 依存: `PH-V06-019`、手動26355点ラン

目的:

反射回数だけでは外周フィールドの実戦寄与を判断できないため、追撃成果の出所を保存します。同時に、快適という所感を実フレーム時間で検証できるラン単位の性能要約を追加します。

受け入れ条件:

- `enemy.hit`が、反射後なら直前の反射面種別と外周面を保持する。
- 障害物 / 外周別の反射後命中・撃破と、外周面別の反射後命中を集計する。
- 両反射面の追撃命中合計が既存の反射後命中と一致し、撃破数が命中数を超えない。
- 旧`RunRecord` v2は追加値0で読み込める。
- ランJSONとCSV / TSVへ、実フレーム標本数、平均、p95、最大、50ms以上のフレーム数、推定FPS、Phaser実測FPSを保存する。
- ゲーム再開時に性能標本を初期化し、ゲームオーバー時の値を確定して自動保存する。
- 15分ソークでp95実フレーム時間34ms以下、50ms超過率1%未満、Phaser実測FPS 15超を確認する。
- 新しい実行時ライブラリを追加せず、アプリ版`0.6.5`、ルール版`phaser-v0.6.4-pulse-ballistics`とする。

対象外:

- 1ランの反射成果を根拠にした武器数値変更。
- WebページのLCP / INP / CLS計測、外部監視サービス、オンライン送信。
- 600秒以降の崩壊バランス変更。
