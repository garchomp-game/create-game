## フェーズ目標

v0.5の目標は「エンドレスの磨き込みとラン記録基盤」です。

- 現在のアリーナを、複数回のプレイテストへ耐えられる主モードにする。
- リザルト、ローカルランキング、履歴によってラン後の振り返りを成立させる。
- 設定、プロフィール、保存の境界を作り、後続のステージ進行や解放へ再利用する。
- タイトル、HUD、演出、音響を、情報理解と手触りが改善する範囲で仕上げる。

v0.5ではゲームプレイの種類を増やすより、今あるゲームを記録し、比較し、再挑戦できる状態を優先します。

状態: 2026-07-11完了。記録、画面、設定、音響の実装、自動検証、15分耐久、通常UIからの3ランと所感反映を完了しました。

## v0.4からの引き継ぎ判断

v0.4は次の判断で閉じ、未実装の操作案をv0.5の着手条件にしません。

| 項目 | 判断 |
| --- | --- |
| 自動射撃 | v0.5の既定操作として暫定採用する |
| 手動射撃 | 左クリックとSpaceを互換操作として維持する |
| 防御ダッシュ | v0.6以降のゲームプレイ試作へ延期する |
| 右クリック固有スキル | アクティブスキル設計と合わせて後続へ延期する |
| 跳弾 | 基盤だけ維持し、武器最終強化または特殊ルールで再評価する |
| 基礎リザルト | v0.4の計測基盤として完了し、v0.5で製品画面へ拡張する |

自動射撃の切り替えは `PH-V05-010` で設定化します。防御行動や固有スキルを先に入れると、記録画面と操作改善の完了判定がぼやけるため、v0.5では扱いません。

## 実装結果

- プレイヤー向け`RunRecord`と版付き保存形式を追加した。
- ランキング対象判定と、手動・デバッグ・自動テストの区分を追加した。
- 履歴とランキングを1つの版付きラン記録envelope内で別コレクションにし、プロフィールと設定はそれぞれ別キーへ保存した。
- 自己記録差、死因、ビルド、シード、対象可否を含む製品向けリザルトを追加した。
- 履歴、ランキング、設定へ移動できるタイトル画面を追加した。
- 32秒BGM、主要効果音8種・15ファイル、音量、ミュート、演出設定を接続した。
- 手動所感を受け、連射や命中などの高頻度SEへ音色とピッチのバリエーションを追加した。
- 開発ログを手動、デバッグ、テストへ分離し、保持件数を設定した。
- 900秒耐久シミュレーションと18状態の画面比較を追加した。

## 実装の分割方針

現在のフォルダ構成で進められるため、大規模な再配置は行いません。ただし、`ArenaScene`へ保存、画面遷移、ランキング判定を追加し続けないよう、必要な責務だけを分けます。

実装結果:

| パス | 責務 | 状態 |
| --- | --- | --- |
| `src/domain/runRecords.ts` | ラン記録、比較キー、対象判定の型 | 分離済み |
| `src/domain/profile.ts` | ゲストプロフィールと設定の型 | 分離済み |
| `src/application/runRecords.ts` | ラン記録生成、順位計算、件数制限 | 分離済み |
| `src/ports/RunRecordStorePort.ts` | 履歴とランキングの保存境界 | 分離済み |
| `src/ports/ProfileStorePort.ts` | プロフィールと設定の保存境界 | 分離済み |
| `src/adapters/storage/LocalRunRecordStore.ts` | `localStorage`実装と復旧 | 分離済み |
| `src/adapters/storage/LocalProfileStore.ts` | ローカルプロフィール実装 | 分離済み |
| `src/adapters/phaser/PhaserMusicController.ts` | BGM状態、ループ、音量 | 分離済み |
| `src/adapters/phaser/PhaserResultPanel.ts` | リザルト表示と操作 | Renderer内に維持。後続候補 |
| `src/adapters/phaser/PhaserTitleScreen.ts` | タイトルとメニュー遷移 | Renderer内に維持。後続候補 |

分割の判断基準:

- 純粋な順位計算、対象判定、データ移行をPhaserへ依存させない。
- `ArenaScene`はラン開始、終了、画面間の調停だけを行う。
- 1回のラン終了イベントから、保存、演出、表示を重複実行しない。
- 空の抽象化や将来だけの階層は作らない。

## ラン記録契約

プレイヤー向け `RunRecord` の最小項目:

- `schemaVersion`
- `id`
- `profileId`
- `capturedAt`
- `modeId`
- `stageId`
- `difficultyId`
- `weaponId`
- `modifierIds`
- `appVersion`
- `rulesetVersion`
- `buildCommit`
- `seed`
- `seedCategory`
- `runOrigin`
- `rankEligibility`
- `elapsed`
- `score`
- `level`
- `kills`
- `damageTaken`
- `lastDamageSource`
- `upgradeRanks`

開発用ランエクスポートには、ウェーブ、エンティティ数、直近イベントなど再現用情報を残せます。ただし、そのままプレイヤー履歴へ保存しません。

## ランキング規則

比較単位:

- `modeId`
- `stageId`
- `difficultyId`
- `rulesetVersion`
- `seedCategory`

エンドレスの並び順:

1. スコア降順。
2. 同点なら生存時間降順。
3. さらに同点なら記録日時昇順。
4. 最後にID昇順。

通常ランキングの対象条件:

- タイトルまたは通常の再挑戦導線から開始した。
- 標準ルールセットを使っている。
- デバッグAPIでHP、XP、敵、時間を変更していない。
- 自動テストではない。
- 固定シードは通常シードと別区分にする。

対象外ランも履歴へ保存できますが、対象外理由をリザルトと詳細画面へ表示します。

## 保存方針

- ラン履歴は最新50件、ランキングは比較単位ごとに上位10件を保持する。
- ラン記録、プロフィール、設定を別キーで扱い、ラン記録内の履歴とランキングは個別に初期化できる。
- 保存形式には版を付け、読込時にZodなどの構造検証を行う。
- 破損項目だけを無視または退避し、アプリ全体を起動不能にしない。
- 保存容量超過や書込失敗は警告として扱い、リザルト表示を継続する。
- 履歴、ランキング、設定、プロフィールを個別に初期化できるようにする。
- 外部アカウントへ進むまで、個人情報や認証情報を保存しない。

## テスト計画

単体テスト:

- ラン出力から `RunRecord` を作る純粋関数。
- ランキング対象判定と対象外理由。
- 並び替え、比較キー、上位10件、直近50件。
- プロフィール、設定、保存形式の検証と破損復旧。
- ラン終了時の重複保存防止。
- BGM状態遷移と演出設定の反映。

E2E:

- 通常ラン、固定シード、デバッグランの保存区分。
- ゲームオーバー後のリザルトと自己記録差。
- 再読込後の履歴、ランキング、設定、プロフィール。
- タイトルから2操作以内のエンドレス開始。
- キーボード、ポインター、カーソル、戻る操作。
- 音量0、揺れ0、点滅0、自動射撃無効。
- 保存破損と保存失敗でも起動できること。

手動確認:

- 短時間、通常、長時間の3本以上を記録する。
- 15分以上でHUD、フレーム、敵数、弾数、ログ容量を確認する。
- リザルト、履歴、ランキングからプレイ差を説明できるか確認する。
- BGMの音量、ループ、集中への影響を確認する。
- 致死時の演出が結果情報を覆わないか確認する。
