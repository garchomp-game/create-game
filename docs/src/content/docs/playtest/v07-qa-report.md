## 判定

`0.7.0` / `phaser-v0.7.0-final-expedition-rc5`を**第10最終遠征の手動再評価用preview候補**とします。既存v0.6.8 productionは変更しません。

RC3 / RC4の手動5勝から、ボスを中央へ寄せて周回すると、貫通による雑魚処理と回復供給を長時間維持できる欠陥が判明しました。RC4はproduction候補として棄却します。RC5はボス戦の回復供給を制御し、遮蔽物か距離で処理する制圧衝撃波と、完遂・速攻ボーナスを追加しました。

自動ゲートは両武器3 seedの6勝、決定論、攻撃可避性、保存、画面回帰、実URLsmokeまで完了しました。production昇格はRC5の通常UIによるPulse / Spread各1本以上の欠陥特化確認まで保留します。自動入力は到達不能と再現性を確認できますが、予告の読みやすさと周回戦術の納得感は代替しません。

## RC3からRC5までの変更

- 敵の初出順を`赤の追跡体 -> 大型の黄色重装体 -> 小型の黄緑高速体 -> 紫の射撃体`へ変更。
- 大型黄は最終遠征だけ基礎HP 3へ2.5倍を適用し、実HP 8へ変更。
- Commanderは最大HP 500、専用HP表示、撃破signalを持つ。
- 最終ボスはHP 3400、段階別速度72 / 96でプレイヤーを追跡する。
- 固定照準斉射は13 / 21発、挟撃護衛は5 / 7体、抑制斉射は9 / 15発。
- ボス戦でも通常ウェーブを継続し、最終ウェーブは出現間隔0.32秒、最大敵64、予算6。
- 「展開地点を確保する」を廃止し、四方侵入と敵役割を示す目的文へ変更。
- EndlessのEX曲線を180 / 1.04 / 360へ変更し、C5をEX Lv17、累計4,258 XPで到達可能にした。
- 最終遠征はステージ定義で180 / 1.12 / 900を維持し、RC3のラン結果を変えない。
- RC5でボス第1 / 第2段階へ半径175 / 220の`command-pulse`を追加。1.35 / 1.05秒の予告後に22 / 34ダメージを与えるが、範囲外か障害物の遮蔽で回避できる。
- ボス戦中の雑魚撃破による回復を決定論的に毎秒1個までとし、抑制数をRunRecordへ残す。
- 勝利時に完遂15,000点と、ボス戦240秒を基準とする最大108,000点の速攻ボーナスを加える。
- expeditionの同点順位は終了時間が短いランを上位にする。Endlessは従来どおり長時間生存を優先する。
- 通常リザルトと遠征リザルトを3列の安定した配置へ変更し、スコア内訳とボス戦時間を重なりなく表示する。

## Endless C5到達性

最終遠征の手動ランでEX C6まで到達した一方、Endlessの長時間観戦では全6ランがC3で終了し、C5のルールが実質的に観測できませんでした。同じ固定3 seed x 2武器、900秒上限でEX曲線だけを比較しました。

| 条件 | C5以上 | 最大EX / C | 平均生存 | 平均スコア |
| --- | ---: | ---: | ---: | ---: |
| RC3既定 180 / 1.12 / 900 | 0 / 6 | Lv12 / C3 | 658.34秒 | 86,649 |
| RC4 Endless 180 / 1.04 / 360 | 3 / 6 | Lv21 / C6 | 683.34秒 | 95,742 |

- C5はEX Lv17から始まり、累計4,258 XPで到達する。
- RC4でもC5以上は3 / 6であり、生存すれば必ず到達する設計ではない。
- 平均生存の増加は25秒、3.8%で、全6ランは900秒前に終了した。
- 最終遠征は従来曲線をStage定義に持つ。RC4で同じ6ランを再実行し、勝敗、終了時刻、スコア、最大密度がRC3と完全一致した。

この比較は「C5が実行可能か」とモード分離を確認する回帰です。観戦AIの生存時間や武器間差を人間プレイの難度保証には使いません。

## 可避性と説明可能性

- 両武器で固定照準斉射を無被弾回避するfixtureが通る。
- ボス追跡は既存の障害物経路を使い、障害物を貫通しない。
- 挟撃増援は安全距離、活動領域、障害物、到達可能性、敵上限を共有する。
- 斉射と護衛抑制射撃は攻撃IDへ帰属し、最後の攻撃種をRun JSONへ残す。
- 通常ウェーブ、護衛、ボス、敵弾は共通上限を使う。
- ボス撃破による勝利、保存、`game.over`は一度だけ発生する。
- command pulseは範囲外または障害物で防げ、開始時点の内外判定と結果を攻撃IDへ帰属する。
- desktopとportraitでCommander 3行バナー、HP 500、ボスHP 3400、広域予告に重なりがない。
- 3列リザルトで通常終了、遠征勝利、長い数値のいずれも文字と操作領域が重ならない。

## 既知事項

- ceiling観戦AIは両武器で6勝したが、人間の武器採否へ直接使わない。
- 最終ボスは道中より明確に高密度になった。第1から第9の初心者向け難度へ流用しない。
- game JSはViteの500KB chunk警告を超える。gzip 475.88KBで、今回の変更による停止条件にはしない。
- `esbuild 0.27.7`相当のLow 1件はWindows開発サーバーの任意ファイル読み取りで、Linux上の静的公開成果物へ直接影響しない。
- 世界観、最終アート、ステージ別ボスBGM、第1から第9、進行保存は未実装である。

## RC5手動採否

過去5ランで道中と両武器の到達性は確認済みです。RC5では変更点に絞り、debug hookと観戦AIを使わず次を行います。

1. PulseとSpreadを各1本以上プレイし、ボス第2段階へ到達する。
2. 少なくとも1本で従来の中央寄せ・近距離周回を意図的に試す。
3. `command-pulse`の予告を認識し、範囲外退避または障害物遮蔽で回避できることを確認する。
4. 雑魚撃破と回復だけで同じ周回を無期限に安定維持できないことを確認する。
5. リザルトで完遂、速攻、合計、ボス戦時間が読み取れ、速い撃破が得点上有利だと理解できることを確認する。

どちらかが第2段階へ届かなかった場合だけ、その武器を1本追加します。重大 / 高の再現不具合が0件で、両武器が第2段階へ到達し、制圧衝撃波と回復制御が意図どおりならproduction昇格候補とします。高難度最終面なので全勝は要求しません。
