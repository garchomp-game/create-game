## 位置づけ

`balanceProbe` は、固定入力モデルと複数シードを使って、ゲームバランスの急な変化を見つける仕組みです。

人間にとっての快適さ、楽しさ、被弾の納得感を表すものではありません。手動プレイの代替にはしません。

## v0.6.2基準値

入力モデル `kiteCollect` の目安です。

| 指標 | 値 |
| --- | ---: |
| 生存時間中央値 | 179.40秒 |
| 毎分撃破数中央値 | 191.30 |
| 毎分スコア中央値 | 2930.10 |
| 初被弾時刻中央値 | 114.13秒 |
| 初強化時刻中央値 | 7.13秒 |
| 到達ウェーブ開始時刻中央値 | 90秒 |
| 回復量中央値 | 80 |
| 回復取得数中央値 | 38 |
| 有効回復取得数中央値 | 7 |
| 最大敵数 | 60 |
| 最大弾数 | 40 |

主要指標はこの値から±20%を回帰検知の目安にします。v0.6.2では障害物対応の敵経路を含めた条件で、5シード・3入力モデルの基準試験と、Pulse / Spread各10シードの比較を実行します。処理量が大きいため、この2試験だけ明示的な15秒・45秒の上限を持ちます。

v0.6.1基準と比べ、`kiteCollect`の初被弾中央値は89秒から114.13秒へ伸びました。射撃体が遮蔽物越しに撃たなくなり、敵が決定論的な迂回路へ集まる影響を含みます。これは自動入力が経路を利用しやすくなった可能性もあるため、序盤を即時に難化せず、人間の手動ランで包囲圧力と被弾の納得感を確認します。

## 使い方

- ゲームルール変更による明確な回帰を検知する。
- 固定シードで変更前後を比較する。
- 敵数、弾数、回復量が許容範囲を外れていないか確認する。
- 敵ナビゲーション追加後も、生存・撃破・初被弾が説明できないほど変わっていないか確認する。
- 手動プレイで見つけた問題を再現する補助に使う。

## 今後の拡張

開始武器、アイテム、装備、ステージ、特殊ルールを追加したら、次を比較できるようにします。

- 武器別の生存時間、毎分撃破数、毎分スコア。
- 最初の挙動変化と最終強化までの時間。
- 戦闘展開カードの連続回数。
- エリートとボスの撃破時間。
- 危険度契約ごとのスコア差。

テスト全体の使い分けは [品質戦略](../../engineering/quality-strategy/) を参照してください。
