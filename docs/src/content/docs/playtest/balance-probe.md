## 位置づけ

`balanceProbe` は、固定入力モデルと複数シードを使って、ゲームバランスの急な変化を見つける仕組みです。

人間にとっての快適さ、楽しさ、被弾の納得感を表すものではありません。手動プレイの代替にはしません。

## v0.6.4基準値

入力モデル `kiteCollect` の目安です。

| 指標 | 値 |
| --- | ---: |
| 生存時間中央値 | 179.37秒 |
| 毎分撃破数中央値 | 196.80 |
| 毎分スコア中央値 | 3034.94 |
| 初被弾時刻中央値 | 84.87秒 |
| 初強化時刻中央値 | 7.07秒 |
| 到達ウェーブ開始時刻中央値 | 90秒 |
| 回復量中央値 | 82 |
| 回復取得数中央値 | 39 |
| 有効回復取得数中央値 | 7 |
| 最大敵数 | 45 |
| 最大弾数 | 44 |

主要指標はこの値から±20%を回帰検知の目安にします。v0.6.4では高弾速Pulseを基準に、5シード・3入力モデルの基準試験と、Pulse / Spread各10シードの比較を実行します。処理量が大きいため、この2試験だけ明示的な15秒・45秒の上限を持ちます。

v0.6.4ではPulseの最大弾速が約1405px/sとなり、終点だけの衝突判定を移動線分上の連続判定へ変更しました。これはPulseだけでなくSpreadの命中精度にも影響するため、v0.6.3との絶対値比較だけで調整せず、同じ版の相対差、固有能力、手動ランを同時に見ます。

固定10シードの中央値は次のとおりです。

| 指標 | Pulse | Spread |
| --- | ---: | ---: |
| 生存時間 | 179.37秒 | 176.40秒 |
| 毎分撃破 | 197.70 | 191.92 |
| 毎分スコア | 3082.61 | 2920.48 |
| 弾単位命中率 | 0.48 | 0.24 |
| 命中斉射率 | 0.36 | 0.63 |
| 命中斉射あたりの異なる敵数 | 1.30 | 1.67 |
| 集束強化命中 / 追加ダメージ | 124 / 24.15 | 0 / 0 |
| 反響回路発動 | 708 | 0 |
| 障害物反射 / 外周反射 | 166 / 517 | 0 / 0 |
| 掃射成立 / 消費 | 0 / 0 | 47 / 46 |

生存、撃破、スコアの差は6%未満で、Spreadの複数標的数はPulseより約28%高い状態です。Pulseは外周反射を中央値517回記録しました。Spreadの掃射は成立47回に対して消費46回で、ラン終了時に未消費チャージが1つ残る許容状態です。自動入力の生存時間は人間の難易度評価ではなく、役割差を壊す変更を検出する回帰値として扱います。

v0.6.5ではルール値を変えず、反射後命中・撃破を障害物と外周へ分けました。`capstoneObstacleFollowUpHits`、`capstoneObstacleFollowUpKills`、`capstoneBoundaryFollowUpHits`、`capstoneBoundaryFollowUpKills`を試行と要約へ追加し、両反射面の追撃命中合計が既存の`capstoneFollowUpHits`と一致することを回帰試験で保証します。

## v0.6.6基準値

外周反射フィールドを再び有効にした現在の`kiteCollect`基準では、生存、毎分撃破、毎分スコア、初被弾、初強化、到達ウェーブはv0.6.4基準を維持します。Pulse強化と外周反射による撃破・ドロップ時系列の変化に合わせ、最大敵数を45、最大弾数を44、回復量を82、有効回復取得数を7へ更新しました。比較用に外周反射を無効化した場合は、それぞれ46、43、106、11を基準にします。各値の許容幅は従来どおり±20%です。

同一3シード・各650秒の旧Pulse再現とv0.6.6比較は次のとおりです。旧再現では基礎ダメージ1.00、対象連続+15%、貫通列倍率0を使います。

| 指標 | 旧Pulse再現 | Pulse v0.6.6 |
| --- | ---: | ---: |
| 平均生存 | 585.46秒 | 612.96秒 |
| 平均スコア | 64992 | 73608 |
| 平均撃破 | 3244 | 3487 |
| 平均XP/分 | 472.99 | 487.73 |
| 集束・貫通追加ダメージ | 893.37 | 2171.34 |

値を変更していないSpreadは同じ3シードで平均648.32秒 / 83193点でした。Pulseとの差は生存5.8%、スコア13.0%で20%基準内です。長時間試験は30fps相当の固定ステップを保ったヘッドレス実行で早送りし、フレームレートを下げて衝突頻度を変える短縮は行いません。

敵ダメージを除く加速長時間入力では、通常ビルド完成が381.43秒でした。合成入力は遠方のXP回収を人間ほど判断できないため、自動有限終了試験の上限だけ7分とし、手動プレイの目標4分から6分は変更しません。v0.6.6の撃破増加でCPU試験の処理量も増えたため、有限終了試験は10秒タイムアウト、7.5秒CPU予算へ更新しました。実ブラウザ性能はラン単位のフレーム統計と15分E2Eで別に判定します。

v0.6.8では外周反射を公開採用し、ルール版だけを`phaser-v0.6.8-pulse-boundary-ricochet`へ分けます。武器・敵・XP・進行の数値と`kiteCollect`回帰値は変えません。旧ルール記録とランキングを分離するための版更新であり、基準値の再調整ではありません。

## 使い方

- ゲームルール変更による明確な回帰を検知する。
- 固定シードで変更前後を比較する。
- 敵数、弾数、回復量が許容範囲を外れていないか確認する。
- 敵ナビゲーション追加後も、生存・撃破・初被弾が説明できないほど変わっていないか確認する。
- 手動プレイで見つけた問題を再現する補助に使う。

## 今後の拡張

開始武器、アイテム、装備、ステージ、特殊ルールを追加したら、次を比較できるようにします。

- 武器別の生存時間、毎分撃破数、毎分スコア。
- 最初の挙動変化と最終強化までの時間。
- 戦闘展開カードの連続回数。
- エリートとボスの撃破時間。
- 危険度契約ごとのスコア差。

テスト全体の使い分けは [品質戦略](../../engineering/quality-strategy/) を参照してください。
