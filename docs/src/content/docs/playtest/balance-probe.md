## 位置づけ

`balanceProbe` は、固定入力モデルと複数シードを使って、ゲームバランスの急な変化を見つける仕組みです。

人間にとっての快適さ、楽しさ、被弾の納得感を表すものではありません。手動プレイの代替にはしません。

## v0.6.3基準値

入力モデル `kiteCollect` の目安です。

| 指標 | 値 |
| --- | ---: |
| 生存時間中央値 | 127.10秒 |
| 毎分撃破数中央値 | 163.34 |
| 毎分スコア中央値 | 2234.30 |
| 初被弾時刻中央値 | 94.93秒 |
| 初強化時刻中央値 | 7.20秒 |
| 到達ウェーブ開始時刻中央値 | 90秒 |
| 回復量中央値 | 80 |
| 回復取得数中央値 | 22 |
| 有効回復取得数中央値 | 6 |
| 最大敵数 | 34 |
| 最大弾数 | 26 |

主要指標はこの値から±20%を回帰検知の目安にします。v0.6.3では単線Pulseを基準に、5シード・3入力モデルの基準試験と、Pulse / Spread各10シードの比較を実行します。処理量が大きいため、この2試験だけ明示的な15秒・45秒の上限を持ちます。

旧Pulseは分裂射撃で最大3発へ増えましたが、v0.6.3は0.12秒の単線連射と集束共鳴へ置き換えています。このため旧基準との単純比較ではなく、同じ版のSpreadとの相対差、固有能力の成立、役割別成果を同時に見ます。

固定10シードの中央値は次のとおりです。

| 指標 | Pulse | Spread |
| --- | ---: | ---: |
| 生存時間 | 127.10秒 | 151.17秒 |
| 毎分撃破 | 163.34 | 175.99 |
| 毎分スコア | 2234.30 | 2515.63 |
| 命中斉射あたりの異なる敵数 | 1.20 | 1.60 |
| 集束強化命中 / 追加ダメージ | 67 / 13.05 | 0 / 0 |
| 掃射成立 / 消費 | 0 / 0 | 23 / 23 |

生存、撃破、スコアの差は20%以内で、Spreadの複数標的数はPulseより15%以上高い状態です。自動入力の生存時間は人間の難易度評価ではなく、役割差を壊す変更を検出する回帰値として扱います。

## 使い方

- ゲームルール変更による明確な回帰を検知する。
- 固定シードで変更前後を比較する。
- 敵数、弾数、回復量が許容範囲を外れていないか確認する。
- 敵ナビゲーション追加後も、生存・撃破・初被弾が説明できないほど変わっていないか確認する。
- 手動プレイで見つけた問題を再現する補助に使う。

## 今後の拡張

開始武器、アイテム、装備、ステージ、特殊ルールを追加したら、次を比較できるようにします。

- 武器別の生存時間、毎分撃破数、毎分スコア。
- 最初の挙動変化と最終強化までの時間。
- 戦闘展開カードの連続回数。
- エリートとボスの撃破時間。
- 危険度契約ごとのスコア差。

テスト全体の使い分けは [品質戦略](../../engineering/quality-strategy/) を参照してください。
