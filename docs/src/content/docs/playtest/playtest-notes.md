## ログ保存

ローカル開発中は、ゲームオーバーになると起点別に保存します。

```text
phaser/logs/runs/*.json   # 手動ラン、最新200件
phaser/logs/debug/*.json  # デバッグラン、最新100件
phaser/logs/tests/*.json  # 明示的な自動テスト、最新20件
```

通常のPlaywright実行と1秒未満のランは、自動で`logs/runs`へ保存しません。プレイヤー向け履歴とランキングは、これらのファイルではなくブラウザの`arena-core.run-records.v2`を使います。

手動ランは毎回異なるシードで始まります。再現したい場合は、URLへ次のように指定します。

```text
?seed=20260619
```

Playwrightと自動テストは固定シードを使います。

## 現在記録できる項目

- シード。
- 設定版。
- 生存時間とスコア。
- 到達ウェーブ。
- 初被弾と初強化の時刻。
- ダメージ源。
- 回復量と回復取得数。
- 選択した強化とランク。
- 武器別の射撃、命中、撃破。

手動で補足する項目:

- ステージID: `default`。
- 特殊ルール: `none`。
- 操作方式: 自動射撃試作。
- 操作負荷と死因の納得感。

## v0.5で追加した項目

- `modeId`
- `stageId`
- `weaponId`
- `modifierIds`
- `rulesetVersion`
- `runOrigin`
- `rankEligibility`
- `appVersion`と`rulesetVersion`の分離。
- `profileId`と設定中の操作修飾子。

強化提示と選択の完全な時系列、意味ある選択が空いた最長時間は、プレイヤー履歴ではなく後続の開発計測として扱います。

## 調整時の注意

- 1回の突出値だけで数値を変えない。
- シード、経過時間、強化構成、ダメージ源を合わせて比較する。
- バランス回帰テストが通っていても、人間の早期死亡や入力負荷を無視しない。
- 高スコアでも緊張感が残り、判断や操作のミスで終わるなら、即時の弱体化は不要とする。
