## v0.6自動比較と手動評価準備

### 2026-07-11 自動比較

固定10シード、180秒、同じ`kiteCollect`入力モデルでPulseとSpreadを比較しました。これは人間の上手さや面白さを代替せず、同条件で役割差と回帰を観測するための値です。

| 指標 | Pulse | Spread |
| --- | ---: | ---: |
| 生存時間中央値 | 165.53秒 | 151.53秒 |
| 毎分スコア中央値 | 2691.26 | 2514.17 |
| 毎分撃破中央値 | 184.20 | 176.99 |
| 弾単位命中率中央値 | 0.18 | 0.19 |
| 命中ボレー率中央値 | 0.31 | 0.52 |
| 命中ボレーごとの異なる敵数 | 1.36 | 1.60 |
| 被ダメージ中央値 | 212 | 301 |
| 危険イベント中の移動距離 | 2537.56 | 1119.39 |

観測:

- SpreadはPulseより命中ボレー率が約68%高く、1命中ボレーの異なる敵数が約18%多い。
- Spreadの被ダメージは約42%多く、広角命中の強みと近距離リスクが同時に出ている。
- Pulse反響回路は10シード中央値で441回発動し、跳弾後命中32回だった。
- 反射後寿命0.12秒版は取得コストに見合わなかったため、0.35秒へ延長した。延長後は生存、毎分スコア、毎分撃破が改善した。
- 射撃体集中は自動入力の生存時間中央値を数%下げ、後半の同時敵数と敵弾数を押し上げた。
- 20%以上の差が出た命中ボレー率と被ダメージは強い役割信号だが、手動操作で楽しいかは未判定である。

## ログ保存

ローカル開発中は、ゲームオーバーになると起点別に保存します。

```text
phaser/logs/runs/*.json   # 手動ラン、最新200件
phaser/logs/debug/*.json  # デバッグラン、最新100件
phaser/logs/tests/*.json  # 明示的な自動テスト、最新20件
```

通常のPlaywright実行と1秒未満のランは、自動で`logs/runs`へ保存しません。プレイヤー向け履歴とランキングは、これらのファイルではなくブラウザの`arena-core.run-records.v2`を使います。

手動ランは毎回異なるシードで始まります。再現したい場合は、URLへ次のように指定します。

```text
?seed=20260619
```

Playwrightと自動テストは固定シードを使います。

## 現在記録できる項目

- シード。
- 設定版。
- 生存時間とスコア。
- 到達ウェーブ。
- 初被弾と初強化の時刻。
- ダメージ源。
- 回復量と回復取得数。
- 選択した強化とランク。
- 武器別の射撃、命中、撃破。
- 強化の全提示、ロック、最大済み、選択時刻、最長選択間隔。
- 命中ボレー率、1射あたりの異なる命中敵数、敵種別成果。
- 最終強化の取得時刻、跳弾回数、跳弾後命中。
- 危険イベントの時系列、区間別移動、出現、撃破、被ダメージ。
- 後半契約の提示、選択、倍率、ランキング除外理由。

手動で補足する項目:

- ステージID: `default`。
- 特殊ルール: `none`。
- 操作方式: 自動射撃試作。
- 操作負荷と死因の納得感。

- `stageId`
- `weaponId`
- `modifierIds`

強化提示と選択の完全な時系列、意味ある選択が空いた最長時間は、プレイヤー履歴ではなく後続の開発計測として扱います。

## 調整時の注意

- 1回の突出値だけで数値を変えない。
- シード、経過時間、強化構成、ダメージ源を合わせて比較する。
- バランス回帰テストが通っていても、人間の早期死亡や入力負荷を無視しない。
- 高スコアでも緊張感が残り、判断や操作のミスで終わるなら、即時の弱体化は不要とする。

## 2026-07-13 長時間安定への所感とv0.6.1対応

手動プレイでは、全通常強化を取得した後に「このままなら任意に長く生存できる」と感じ、最後は意図的にプレイを崩して終了したという所感が得られました。難易度上昇自体は感じられた一方、完成後の成長選択と戦闘展開が止まり、安定経路を崩す仕組みが不足していました。

この所感を根拠に、次をv0.6.1へ実装しました。

- 通常ビルド後のEXレベルと反復限界強化。
- 240秒以降、45秒ごとに上がり続ける脅威ティア。
- 射撃体集中、高速群襲来、重装包囲の反復デッキ。
- 600秒以降のアリーナ崩壊と領域外ダメージ。
- EX、脅威、イベント、崩壊のランJSONと表形式要約。

次の手動ランでは、スコアの高さだけでなく次を記録します。

- 通常ビルド完成時刻と最初のEX取得時刻。
- EXで選択に迷った候補と、その理由。
- 危険イベントごとに変えた移動または標的優先。
- 敵が単調に硬いと感じ始めた時刻。
- 崩壊の予告を理解できたか、危険領域から戻れたか。
- 最終死因と崩壊段階が納得できたか。
