## 現在の状態

Phaser版v0.6.8は、Phaser 4.2.1 WebGL、可変桁HUD、ゲスト・ローカル保存を公開ベータとして固定した版です。[Cloudflare公開URL](https://arena-core.garchomp-game.workers.dev/)は基準commit `ff686f992a65`、Version ID `e86f90b8-ea15-4d1d-b01b-59e4f9fea78e`を配信しています。タイトルとリザルト、HTML meta、ベータ情報ページからアプリ版、ルール版、ビルドを識別できます。保存データ、削除、既知制約、フィードバック、第三者ライセンスも同一オリジンで公開します。

v0.7 RC6は[フォローアップ監査](../../playtest/v07-rc6-integration-report/)を受け、ボス遭遇の無期限signal待ち、全run終端、390秒境界、配置期限と予告方向、profile別PB、0.01秒比較、ランキングboard、probeのnormal / repair分離を修正しました。Endlessの戦闘ルールは変えていないため、既存`phaser-v0.6.8-pulse-boundary-ricochet` PBを維持します。2400 HP有限回復候補は棄却し、通常UIのPulse 2本・Spread 1本で中央周回と回復循環を再確認した結果、既定controlをproduction候補として採用しました。production trafficはリポジトリ統合とUI候補の採否までv0.6.8のまま維持しています。

ゲーム数値はv0.6.6のままです。672.20秒 / 92728点でアリーナ崩壊2段階へ到達したPulse手動ランにより、精密射撃調整の採用を完了しました。Pulseは照準・貫通・反射角で成果上限を伸ばす単線武器、Spreadは広角の複数標的処理を安定させる武器として分かれています。

アリーナ内の4障害物とPulse外周反射フィールドを公開ルールとして採用しました。直近のPulse / Spread各2手動ランは約2.5万点から3.3万点、約350秒から417秒へ重なり、Pulseだけが外周追撃1332命中・625撃破を記録しています。外周反射はSpreadを直接強化せず、Pulseの貫通順と復路射線を成果へ変えるため、数値を変えず有効のまま固定します。ルール版は`phaser-v0.6.8-pulse-boundary-ricochet`へ分け、旧記録を新ランキングへ混ぜません。

`experiment/auto-pilot-observer`では観察専用AIを試験しています。`?autopilot=pulse`または`?autopilot=spread`で直接開始し、プレイ中は`O`でAIと手動を切り替えます。v3は相対速度と連続TTCによる敵弾・接触回避、障害物経路を含む回収ETA、目標コミット、到着時の回復価値、Pulseの到達可能な貫通列、Spreadの扇形複数標的照準を扱います。訪問履歴巡回は`VITE_ARENA_AUTO_PILOT_PATROL_STRATEGY=visit-history-v1`で検証できますが、開発比較で採用条件を満たさなかったため既定は`periodic-v3`です。候補ランには`auto-pilot:patrol:visit-history-v1`を追加し、通常のAIランと区別します。AIランはランキング対象に入りません。詳細は[観戦AIの設計判断](../../engineering/auto-pilot-architecture/)を参照してください。

## v0.6.8で追加した境界

- Pulse外周反射を公開採用し、機能フラグを有効のまま固定した。武器・敵・XP・進行数値は変更しない。
- ルール版を`phaser-v0.6.8-pulse-boundary-ricochet`へ更新し、旧ルール記録とランキング比較条件を分離した。
- タイトルとリザルト、HTML metaへアプリ版、ルール版、12桁build commitを表示する。
- タイトルからベータ情報を開き、ローカル保存、外部送信なし、全データ削除、既知制約、報告テンプレート、第三者ライセンスを公開する。
- production検査で開発ログAPI、debug hook、fixture、ソースマップ、ログ、レビュー資料の混入を拒否する。
- Chromeの960 x 540、390 x 844とFirefoxの960 x 540で、WebGL、開始操作、版情報、ベータ情報、全データ削除をsmoke testする。

## v0.6.7で追加した境界

- Phaserを`3.90.0`から`4.2.1`へ更新し、ESM importをnamespace形式へ統一した。
- WebGLを既定rendererとし、v3互換の`roundPixels: true`を明示した。Canvasは環境変数による診断用だけに残す。
- E2EではWebGLの画面読取を安定させる設定を有効にするが、本番では無効にする。
- HPとXPをラベル / 値の行と独立したバーへ分け、長時間のHP、EX、経験値、5桁スコアを画像回帰へ追加した。
- 遭遇バナーと画面外敵表示は新しいHUD占有領域を避ける。
- アプリ版だけを`0.6.7`へ更新し、ルール版`phaser-v0.6.6-pulse-precision-tuning`と保存形式は維持した。

## v0.6.6で追加した境界

- Pulseの基礎ダメージを1.00から1.16へ上げ、基礎射撃間隔0.12秒と弾数1を維持する。
- 集束共鳴を1段階+20%、最大4段階+80%とする。
- 跳弾前の同じPulse弾が複数敵を貫通した場合、先に直撃した敵数を集束段階として追加ダメージへ加える。
- 跳弾後は従来どおり集束対象外とし、貫通列ボーナスも適用しない。
- ランJSONとCSV / TSVでは、対象連続と貫通列の強化命中数・追加ダメージを分ける。旧記録は追加4項目を0で読む。
- Spread、敵HP、XP、脅威、危険イベント、崩壊、外周反射フィールド無効は変更しない。
- 観戦AI比較は30fps相当の決定論的ヘッドレス実行を使い、衝突頻度を変える低fps早送りは採用しない。

## v0.6.5で追加した境界

- 障害物反射後と外周反射後の命中・撃破を分け、外周追撃命中は左・右・上・下にも帰属する。
- JSON、`summary.tsv`、`summary.csv`へ、フレーム標本数、平均 / p95 / 最大の実フレーム時間、50ms以上のフレーム数、推定FPS、Phaserの`actualFps`を出力する。
- フレーム時間はシミュレーションへ渡す50ms上限値ではなく、Phaser `TimeStep.rawDelta`の観測値を使う。
- ルール値とランキング比較条件は変えず、アプリ版だけを`0.6.5`へ更新した。ルール版は`phaser-v0.6.4-pulse-ballistics`を維持する。
- ゲームループ性能にはPhaserの既存TimeStepを使い、ページ表示指標向けライブラリや長時間タスク監視の追加依存は導入しない。

## v0.6.4で追加した境界

- 加速弾はPulseだけ1ランク22%、Spreadと開発用Pierceは従来どおり15%とした。5ランク時はPulse約1405px/s、Spread約965px/sになる。
- 弾の移動線分と敵円を連続判定し、50msステップでも高速弾がFastをすり抜けない。貫通は線分上の交差時刻順に処理する。
- 反響回路取得後のPulseは、障害物または固定アリーナ外周で合計1回反射する。外周は敵、敵弾、プレイヤー、画面外スポーン、崩壊領域へ作用しない。
- 反射前後の線分を同じフレームで処理し、往路の直接命中と復路の跳弾後命中を分離して記録する。
- 反響回路取得中は外周へシアンの反射フィールドを表示し、外周反射地点には障害物反射と区別できる短い演出を出す。
- 障害物反射、外周反射、左・右・上・下の面別反射をランJSON、履歴、CSV / TSVへ保存する。旧v2記録は既定値0で読み込む。
- 15分耐久で射撃体集中時の同時弾が408へ達したため、通常の攻撃頻度を保ったまま同時敵弾だけを256で制限する安全上限を追加した。
- 競技条件を分けるため、ルール版を`phaser-v0.6.4-pulse-ballistics`へ更新した。

## v0.6.3で追加した境界

- Pulseの分裂射撃を廃止し、0.12秒の単線連射と、同じ敵への直撃で最大4段階まで追加ダメージを蓄積する「集束共鳴」へ置き換えた。
- Spreadだけが分裂射撃で3発から5発へ成長する。最終強化「掃射循環」は1斉射で異なる3体へ命中すると、次弾までの残り待ち時間を70%へ短縮する。
- 反響回路はPulse専用、掃射循環はSpread専用とし、両武器とも通常24ランクと最終強化1ランクの合計25ランクでEXへ移る。
- 240秒以降のHP倍率をFast 1.10、Chaser 1.12、Ranged 1.15、Brute 1.17へ分け、序盤HPを変えず後半の優先標的を作る。
- 武器、通常強化、EX強化、後半契約をCanvas文字からDOMの`button`へ移した。縦画面では親要素全体を使って1列表示し、Tab、Enter、数字キー、ポインターを維持する。
- 集束の強化命中、追加ダメージ、最大段階、敵種別撃破と、掃射の成立、消費、最大同時標的をランJSON、履歴、CSV / TSVへ保存する。旧v2記録は既定値0で読み込む。
- 競技条件を分けるため、ルール版を`phaser-v0.6.3-weapon-identities`へ更新した。

## v0.6.2で追加した境界

- 敵からプレイヤーまで射線が通る場合は従来どおり直線追跡し、障害物で塞がれる場合だけ共有Dijkstra経路場を使う。
- 経路場はプレイヤーのグリッド位置と敵半径ごとに作り、同条件の敵で共有する。最近使った目標位置は最大64件再利用する。
- 経路セルは敵半径と2pxの余白で膨張し、実際の直線移動は敵半径で判定する。経路が得られない場合は従来移動へ戻す。
- 射撃体は遮蔽物越しに発射せず、射線が取れる位置まで経路追跡する。
- EXは4候補を1サイクルとし、取得した候補をサイクル終了まで除外する。残り1候補は自動取得する。
- 過給サイクルは1段階+10%・最大5段階、超過駆動は1段階+6%・最大5段階とする。限界出力と増設コアは上限を持たない。
- EXサイクルと自動取得を履歴・JSON・TSV / CSVへ、敵の直接追跡・迂回・フォールバック・経路場構築回数を詳細JSONへ出力する。
- 競技条件を分けるため、ルール版を`phaser-v0.6.2-navigation-extra-cycle`へ更新した。

## v0.6.1で追加した境界

- 通常強化がすべて最大になると、通常ランクを維持したままEXレベルへ移る。
- EX強化は攻撃力と最大HPを無制限、連射と移動を安全上限付きで反復取得できる。上限到達済みの無効果候補は表示しない。
- 脅威ティアは240秒から45秒ごとに増え、敵HP、攻撃力、スコアは上限なく上がる。
- 敵速度、同時敵数、出現頻度、敵弾速度は、画面の可読性と性能を守る上限を持つ。
- 射撃体集中、高速群襲来、重装包囲を、同一シードで再現可能なシャッフルバッグから反復する。
- 600秒から安全領域を45秒ごとに縮小し、領域外ダメージと最終的な安全領域消失で有限終了を作る。
- EX、脅威、崩壊、イベント回数を、リザルト、履歴、ランJSON、`summary.tsv`、`summary.csv`へ出力する。
- 競技条件を分けるため、ルール版を`phaser-v0.6.1-endless-escalation`へ更新した。

## v0.6で追加した境界

- タイトルからPulseまたはSpreadを選び、同じ武器とシードで再挑戦できる。
- Pierceは通常選択へ出さず、固定配置・開発計測用として維持する。
- 強化定義を小さなカタログへ分け、分類、対象武器、解放条件、効果をUI・シミュレーション・計測で共有する。
- 通常XPは`growth: 1.25`、上限120とし、全強化後は空の通常候補を出さずEX進行へ移る。
- Pulseの武器系7ランクで反響回路を解放し、跳弾と跳弾後命中を記録する。
- 出現、強化、ドロップ、戦闘展開、ステージ候補の乱数列を分離する。
- 射撃体集中を最初の予兆付きイベントとして表示・記録する。
- 過負荷契約は敵速度1.12倍、スコア1.3倍とし、標準外ルールとしてランキングから除外する。
- デバッグAPI、fixture、開発ログPOST、ラン出力組み立てを`ArenaScene`外へ抽出した。

## v0.5で実装した境界

- プレイヤー向け`RunRecord`と開発診断用ランJSONを別の型、別の保存先へ分けた。
- `appVersion`、`rulesetVersion`、`buildCommit`の取得元を集約した。
- 手動、デバッグ、自動テストと、ランダム / 固定シードを区分した。
- デバッグと自動テストは履歴へ残せるが、通常ランキングから除外する。
- ゲームオーバー1回につき1件だけ保存し、破損や容量超過でもリザルトを表示する。
- ゲストプロフィール、設定、ラン記録を別キーにし、履歴とランキングはラン記録内で個別に初期化する。
- タイトルから開始、ランキング、履歴、設定へキーボードとポインターで移動できる。
- メニューでは通常カーソル、プレイ中は照準カーソルを使う。
- BGMと効果音を分離し、ユーザー操作前の音声ロックを待って再生する。

## プレイと耐久確認

v0.5の通常UIから、同じビルド`34206f74bd8a`で次の5ランを記録しました。ブラウザ履歴、ランキング、開発ログの内容が一致し、すべて通常ランキング対象です。

- 短時間: 3211点 / 約98秒 / レベル11。
- 通常: 8196点 / 約178秒 / レベル14。
- 長め: 14172点 / 約272秒 / レベル16。
- 音響更新後1: 16907点 / 約301秒 / レベル16。
- 音響更新後2: 18742点 / 約322秒 / レベル16。
- 操作とUIは良好と評価された。初版音響は単調という所感を受け、SEへ2〜3バリエーション、BGMへ32秒・4区間の展開を追加した。
- 自動耐久: 900秒、30fps相当の27,000ステップを完走。敵、弾、ピックアップ、座標は設定した上限内だった。実時間ブラウザ確認の代替ではない。
- 実時間ブラウザ耐久: 15.1分を完走。終盤FPS、JSヒープ、個体数、Canvas画素、コンソールエラーの判定を通過した。

旧v0.4ログ5件は長時間傾向の参考として残しますが、`rulesetVersion`を持たないためv0.5の履歴やランキングへ移行しません。詳細は[手動プレイ記録](../../playtest/playtest-notes/)を参照してください。

v0.6.3の手動比較では、Pulse 3本が4689点 / 121秒、8250点 / 171秒、6908点 / 146秒で通常ビルド完成へ届かず、Spreadは34434点 / 421秒、277秒で通常ビルド完成、EX Lv5 / C2へ到達しました。分裂を失ったPulseの難度上昇と、Spreadの安定した群れ処理が明確だったため、Pulseへ分裂を戻さずv0.6.4の高弾速と外周反射で技能上限を広げました。

v0.6.4の固定10シード比較では、Pulse / Spreadの生存時間中央値は179.37秒 / 176.40秒、毎分撃破は197.70 / 191.92、毎分スコアは3082.61 / 2920.48で、差はすべて6%未満でした。命中斉射あたりの異なる敵数は1.30 / 1.67でSpreadが約28%多く、Pulseは障害物反射166回、外周反射517回を中央値で記録しています。連続衝突判定はSpreadにも命中判定の正確さとして効くため、Spreadの数値は変更せず手動比較で監視します。

v0.6.4のPulse手動再評価は7972点 / 163.03秒と26355点 / 353.96秒でした。長いランは274.79秒で通常ビルドを完成しEX Lv3へ到達、毎分撃破278.16、毎分スコア4467.41、弾単位命中率43.87%でした。障害物1712回、外周3662回の反射と1142回の反射後命中を記録しています。同じ版の短いランとの振れ幅と、集中した立ち回りで伸ばせた所感から、Pulseを技能寄り武器として採用し、数値は変更しません。

## 残る課題

既知課題:

- Phaser表示、session / run lifecycle、debug / AI / performanceの分離は完了した。RC5現在は`ArenaScene` 653行、World View 863行、Screen View 241行、純粋Presenter 450行である。次は画面別Presenter、DOM選択のデータ整形、theme tokenを実変更に合わせて分ける。
- Phaserを含むRC5の本番JSは約1.75MB、gzip約476KBで、Viteの500KB警告が残る。
- `npm audit`はVite経由の`esbuild 0.27.7`に、Windows開発サーバー限定のLow 1件を報告する。公開成果物への直接影響はないが、互換範囲内の修正版が入る時点で更新する。
- 外部無料BGMの採用は必須ではない。追加選曲する場合は、現在の生成音源と比較し、再配布、加工、クレジット条件を資産台帳へ記録する。
- 外部ログイン、オンラインランキング、クラウド同期は意図的に対象外である。
- v0.6.6のPulse精密射撃調整は自動比較と92728点の手動採否を完了した。単発スコアを保証値にはせず、公開ベータでPulse / Spreadの複数ラン分布を観測する。
- EX循環、敵経路、敵役割別HP、イベント間隔、収縮開始時刻は自動保証済みである。600秒以降の崩壊体感と熟練プレイでの終了時刻は、公開ベータの非停止条件として観測を続ける。
- 固定入力の初被弾中央値はv0.6.4基準で84.87秒となった。自動入力は人間の地形利用や反射角を再現しないため、序盤圧力は手動所感を見て判断する。
- 無敵時間中に重なった敵弾はダメージを与えず消滅するルールで固定した。

v0.6の手動採否、外周反射の公開固定、長時間HUD、Phaser 4.2.1 WebGL移行、production耐久、公開情報、Cloudflare配信は完了しました。公開URLの通常UIだけを使ったスモークで、設定保存、ランキング、履歴、Pulse開始、自然終了とRunRecord保存、リトライ、一時停止、タイトル復帰、ベータ情報、第三者ライセンスを通過しました。保存記録は`0.6.8`、`phaser-v0.6.8-pulse-boundary-ricochet`、`ff686f992a65`で一致し、自動ブラウザ由来のためランキング対象外になりました。実装不具合の未解決P0はありません。
